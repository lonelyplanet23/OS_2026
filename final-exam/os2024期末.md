## 四、死锁问题（本题共10分，第1小题5分，第2小题5分）

一个系统有4个进程（A、B、C、D）和5种可分配资源（R1、R2、R3、R4、R5）。进程只有在其所有资源的最大需求量得到满足后才会释放资源。

当前进程的资源分配情况和进程最大资源需求如下表：

| | 已分配资源 | | | | | 最大需求量 | | | | |
|---|---|---|---|---|---|---|---|---|---|---|
| | R1 | R2 | R3 | R4 | R5 | R1 | R2 | R3 | R4 | R5 |
| 进程A | 1 | 0 | 2 | 1 | 1 | 1 | 1 | 2 | 1 | 3 |
| 进程B | 2 | 0 | 1 | 1 | 0 | 2 | 2 | 2 | 1 | 0 |
| 进程C | 1 | 1 | 0 | 1 | 0 | 2 | 1 | 3 | 1 | 0 |
| 进程D | 1 | 1 | 1 | 1 | 0 | 1 | 1 | 2 | 2 | 1 |

当前系统可用资源为（表中x≥0）：

| R1 | R2 | R3 | R4 | R5 |
|----|----|----|----|----|
| 0 | x | 1 | 1 | 2 |

**问题：**
1. 该状态是否是安全状态？给出理由。
2. 如果此时A进程请求2个R5资源，为了确保不产生死锁，并可以接受此请求，x的取值最小是多少？

### 解答

**(1) 当前需求矩阵：**

| | 已分配资源 | | | | | 需求矩阵 | | | | |
|---|---|---|---|---|---|---|---|---|---|---|
| | R1 | R2 | R3 | R4 | R5 | R1 | R2 | R3 | R4 | R5 |
| 进程A | 1 | 0 | 2 | 1 | 1 | 0 | 1 | 0 | 0 | 2 |
| 进程B | 2 | 0 | 1 | 1 | 0 | 0 | 2 | 1 | 0 | 0 |
| 进程C | 1 | 1 | 0 | 1 | 0 | 1 | 0 | 3 | 0 | 0 |
| 进程D | 1 | 1 | 1 | 1 | 0 | 0 | 0 | 1 | 1 | 1 |

可用资源：(0, x, 1, 1, 2)

即便x=0，也存在安全序列 **D → A → C → B**，使得所有进程能够执行完毕：

- 进程D可以分配资源运行，当它结束时，可用资源向量为(1, 1, 2, 2, 2)
- 进程A可以分配资源执行，当A结束时，可用资源向量为(2, 1, 4, 3, 3)
- 进程C可以被满足，当C执行结束后，可用资源向量为(3, 2, 4, 4, 3)
- 最后进程B运行完毕

**结论：该状态是安全的。**

**(2) 如果接受A进程分配2个R5资源的请求：**

新的资源分配情况：

| | 已分配资源 | | | | | 需求矩阵 | | | | |
|---|---|---|---|---|---|---|---|---|---|---|
| | R1 | R2 | R3 | R4 | R5 | R1 | R2 | R3 | R4 | R5 |
| 进程A | 1 | 0 | 2 | 1 | 3 | 0 | 1 | 0 | 0 | 0 |
| 进程B | 2 | 0 | 1 | 1 | 0 | 0 | 2 | 1 | 0 | 0 |
| 进程C | 1 | 1 | 0 | 1 | 0 | 1 | 0 | 3 | 0 | 0 |
| 进程D | 1 | 1 | 1 | 1 | 0 | 0 | 0 | 1 | 1 | 1 |

可用资源：(0, x, 1, 1, 0)

- 如果x=0：无法满足任何进程需要，不存在安全序列，状态不安全
- 如果x=1：存在安全序列 **A → D → B → C**
  - A可执行，结束时可用资源向量为(1, 1, 3, 2, 3)
  - D可执行，结束时可用资源向量为(2, 2, 4, 3, 3)
  - B可执行，结束时可用资源向量为(4, 2, 5, 4, 3)
  - C可执行

**结论：x的最小取值为1。**

## 五、内存管理（本题共15分）

一个32位虚拟存储系统采用两级页表结构，每个页面大小为4096字节。该系统逻辑地址中，第22到31位是第一级页表（页目录）的索引，第12位到21位是第二级页表的索引，页内偏移占第0到11位。

每个页表（或目录）项包含20位物理页框号和12位标志位（第0位为1时表示该表项有效)。

假设逻辑地址0x00808A00对应的第一级页表和第二级页表的起始物理地址分别为0x00400000和0x00401000。

### 问题

1. 该系统的整个逻辑地址空间一共有多少个页面？整个二级页表有多少页表项？

2. 请找出逻辑地址0x00808A00对应的第一级页表项的物理地址和第二级页表项的物理地址。请找出逻辑地址0x12345678对应的第一级页表项的物理地址。

3. 在保证逻辑地址0x00808A00映射到物理地址0x00808A00的情况下，写出逻辑地址0x00808A00对应的第一级页表项和第二级页表项的内容。

4. 如果从0xA0000000开始映射4MB大小页表空间，求第一级页表（页目录）的起始逻辑地址和页目录自映射表项的逻辑地址？

### 解答

1. **页面数量：** 2^20 = 1M个页面
   **页表项数量：** 2^20 = 1M个页表项

2. **0x00808A00地址解析：**
   - 第一级页表索引：0x00808A00 >> 22 = 0x2（10位）
   - 第二级页表索引：(0x00808A00 >> 12) & 0x3FF = 0x8（10位）
   - 第一级页表项物理地址：0x00400000 + 0x2 × 4 = **0x00400008**
   - 第二级页表项物理地址：0x00401000 + 0x8 × 4 = **0x00401020**

   **0x12345678地址解析：**
   - 第一级页表索引：0x12345678 >> 22 = 0x48
   - 第一级页表项物理地址：0x00400000 + 0x48 × 4 = **0x00400120**

3. **页表项内容：**
   - 第一级页表项：第二级页表物理页框号 + 标志位 = 0x00401 << 12 | 0x001 = **0x00401001**
   - 第二级页表项：目标物理页框号 + 标志位 = 0x00808 << 12 | 0x001 = **0x00808001**

4. **页目录自映射：**
   - 页目录起始逻辑地址：0xA0000000 | (0xA0000000 >> 10) = **0xA0280000**
   - 页目录自映射表项地址：0xA0000000 | (0xA0000000 >> 10) | (0xA0000000 >> 20) = **0xA0280A00**

## 八、进程同步与互斥（本题共15分）

某校开发了一个在线选课系统，系统提供课程查询和选课操作，每门课程都设置了最大容量。

为了避免大规模并发访问造成的系统拥塞，最多允许N个用户同时进入系统(maxVisit = N)。

用户通过查询操作(browse_course)或选课操作(select_course)分别对课程的剩余容量（quota）进行读或写。

为了简化问题，假定系统中只有一门课程，最大容量为M。

**要求：**
1. 用户按照到达顺序对课程剩余容量等信息进行读/写
2. 当多个查询操作连续到达时，应允许并发读取课程剩余容量信息
3. 当有一个正在进行的选课操作时，不允许执行其他选课、查询操作
4. 当课程剩余容量为0时，不允许执行选课操作，否则执行选课操作（quota = quota - 1）

**请基于PV操作写出browse_course和select_course之间的正确同步/互斥关系，给出所使用信号量的定义。**

### 解答

```c
// 共享变量
int readcount = 0;           // 读者计数
int quota = M;               // 课程剩余容量

// 信号量定义
Semaphore maxVisit = N;      // 限制同时进入系统的用户数
Semaphore rmutex = 1;        // 保护readcount的互斥锁
Semaphore mutex = 1;         // 保护quota的读写互斥锁
Semaphore in = 1;            // 保证用户按到达顺序执行的互斥锁

main() {
    Cobegin
        select_course();
        browse_course();
    Coend
}

select_course() {
    P(maxVisit);             // 限制同时进入系统
    P(in);                   // 按到达顺序排队
    P(mutex);                // 互斥访问quota
    if (quota > 0) {
        quota--;             // 选课成功，剩余容量减1
    }
    V(mutex);
    V(in);
    V(maxVisit);
}

browse_course() {
    P(maxVisit);             // 限制同时进入系统
    P(in);                   // 按到达顺序排队
    P(rmutex);               // 互斥访问readcount
    if (++readcount == 1) {
        P(mutex);            // 第一个读者锁住写操作
    }
    V(rmutex);
    V(in);
```

## 九、文件系统（本题共5分，第1小题3分，第2小题2分）

### 问题

(1) 在基于索引结构的文件系统中，目录项中一般不保存子目录或文件数据所在的物理块指针，而是保存索引节点所在的物理块指针。假设根目录的索引节点已读入内存，如果用户要读取文件`/foo/bar`，请简要分析文件系统的主要处理过程。

(2) 在索引文件系统中，如果磁盘块大小为512个字节，块地址占4字节，在索引节点中使用8个直接块地址、4个一次间接块地址和1个二次间接块地址，那么可以访问的文件最大为多少KB？

### 解答

**(1) 文件读取过程：**

1. 根据根目录的索引节点，得到根目录内容所在的块地址
2. 读取根目录内容，找到"foo"目录项，得到foo的索引节点所在的块地址
3. 读取foo的索引节点，得到/foo目录内容所在的块地址
4. 读取/foo的目录内容，找到"bar"文件项，得到/foo/bar的索引节点所在的块地址
5. 读取/foo/bar的索引节点，得到文件bar的数据所在的物理块号
6. 根据索引节点中的块地址，读取文件bar的数据内容

**(2) 最大文件大小计算：**

每个磁盘块可存放的块地址数 = 512字节 / 4字节 = 128个

- 直接块：8 × 512字节 = 4KB
- 一次间接块：4 × 128 × 512字节 = 256KB
- 二次间接块：1 × 128 × 128 × 512字节 = 8192KB

**最大文件大小 = 4 + 256 + 8192 = 8452 KB**
