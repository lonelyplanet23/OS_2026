## 🎯 核心听课重点（必须认真听，考点+面试考点）

### 1️⃣ 覆盖 vs 交换 概念辨析（选择题/简答题常考）

| 对比维度 | 覆盖 | 交换 |
|----------|------|------|
| **谁负责划分** | 程序员手动划分 | 操作系统自动完成 |
| **对程序员透明** | ❌ 不透明（需要程序员感知） | ✅ 透明（程序员无需关心） |
| **核心思想** | 不同时使用的代码/数据共享同一块内存 | 暂时不用的进程整体换出到外存 |
| **目的** | 让大程序能在较小内存中运行 | 增加并发运行的程序数目 |

**关键结论**：
- 一个覆盖段中包含多个不会同时使用的模块，**覆盖区的大小等于最大模块的大小**
- 覆盖和交换是虚拟存储思想的前身，都是通过时间复用空间来突破物理内存限制

---

### 2️⃣ 局部性原理（必考，虚拟存储的理论基础）

虚拟存储能有效工作，根本原因就是**程序运行具有局部性原理**。

- **时间局部性**：最近访问过的指令/数据，很短时间内很可能再次访问
- **空间局部性**：最近访问过的单元，其附近单元很可能很快被访问

#### 经典案例：矩阵乘法循环顺序对性能的影响

两种写法：
- **IJK版本**：按数学定义，外层循环i行j列，内层k累加 → 运行时间约 **11秒**
- **KIJ版本**：外层k，内层i,j，累加直接写到目标矩阵 → 运行时间约 **5秒**

**为什么KIJ更快？**
- 最内层循环按**行连续访问**，符合空间局部性
- CPU Cache按Cache Line缓存，一次加载一整行，后续访问都命中Cache
- IJK版本跳跃访问，Cache命中率低，频繁访问主存 → 慢

#### 引申：排序算法的局部性
- **快排**：局部性好 → 实际运行比堆排快
- **堆排**：堆调整频繁跳跃访问，局部性差 → 虽然最坏复杂度O(nlog n)，实际更慢
- **STL排序**：短数组用插入排序（O(n²)但局部性好），大数组用快排

### 3️⃣ 虚拟存储的核心特征（简答题）

**四个字记住**：**按需调页**

| 特征 | 含义 |
|------|------|
| **多次性** | 程序不需要一次性全部装入，分多次调入内存 |
| **对换性** | 运行过程中允许换入换出，不用的页换出到外存，需要的页换入 |
| **虚拟性** | 逻辑上扩充内存容量，给进程提供巨大的连续地址空间 |

**重要结论**：
- 虚拟存储的**最大容量由地址结构（CPU地址总线位数）决定**，与物理内存和外存容量无关
- 32位系统最大虚拟地址空间是4GB，64位系统几乎不限
- 虚拟存储对**应用程序员透明**，由操作系统软硬件共同实现

---

### 4️⃣ Cache vs 虚拟存储对比（简答题常考）

二者都利用了局部性原理，但解决的问题不同：

| 对比维度 | Cache | 虚拟存储 |
|----------|-------|----------|
| **解决问题** | **速度差** → 匹配CPU和内存的速度差异 | **容量问题** → 用外存扩充内存容量 |
| **不命中开销** | 小（十几~几十个时钟周期） | 大（毫秒级磁盘IO），差几个数量级 |
| **对谁透明** | 硬件实现，对操作系统透明 | OS实现，对应用程序员透明 |
| **存储层次** | CPU <-> 内存 | 内存 <-> 外存 |

---

### 5️⃣ 请求分页与纯分页的区别

- **纯分页（实存）**：要求程序**全部一次性装入**内存才能运行
- **请求分页（虚存）**：只装入当前需要的少数页面，运行中缺页时再动态调入 → **真正的虚拟存储**

**页表需要新增的内容**：
- **存在位（Present/Absent位）**：标记该页是否已在物理内存
- **修改位、访问位**：用于页面置换
- **外存地址**：该页不在内存时，记录它在外存的位置

---

### 6️⃣ 32位X86 Linux进程地址空间布局

| 地址范围 | 用途 | 说明 |
|----------|------|------|
| 0 ~ 0x08048000 | 低地址禁止区域 | 捕获空指针访问，访问这里直接报错 |
| 0x08048000 ~ 3GB | **用户空间** | 代码段、数据段、BSS、堆、栈 |
| 3GB ~ 4GB | **内核空间** | 每个进程都映射这部分，减少上下文切换时TLB刷新 |

**设计思想**：将内核代码映射到每个进程地址空间，系统调用时不用切换地址空间，减少TLB刷新开销。

## 核心知识主线

```
突破物理内存限制
    ├─ 早期方法：覆盖（程序员手动）、交换（OS自动）
    ├─ 理论基础：局部性原理（时间+空间）
    ├─ 现代实现：虚拟存储（按需调页）
    │   ├─ 特征：多次性、对换性、虚拟性
    │   ├─ 硬件支持：缺页中断
    │   └─ OS需要处理：调页时机、替换策略
    └─ 对比：Cache解决速度，虚存解决容量
```
