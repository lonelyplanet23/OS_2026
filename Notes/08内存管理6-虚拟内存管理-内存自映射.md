# 📚 《操作系统》核心Notes：虚拟内存 - 页目录自映射

**本质：把上一级页表存放在下一级页表所在的空间中，放在同一段连续的地址空间**
**然后，只需要通过计算（地址转换），就可以由页表基地址找出页目录表、页目录表项**
## 模块一：引入背景 —— 操作系统如何管理页表？
**1. 多级页表的空间占用回顾**
*   在32位系统、4KB页面大小下，4GB地址空间需要 1M（$2^{20}$）个页表项（PTE）。每个PTE占4字节，因此**完整的页表需要 4MB 的连续虚拟地址空间**。
*   为了管理这 4MB 的页表，系统引入了**页目录（Page Directory）**。4MB页表被分成 1024 个页表页，页目录占 1 页（4KB），包含 1024 个页目录项（PDE），每一项对应 4MB 内存。

**2. 痛点：OS自己如何修改页表？**
*   页表是用来映射内存布局的。当操作系统需要修改映射关系时，它**必须写页表所在的内存**。
*   操作系统使用的是**虚拟地址**，因此这 4MB 的页表（包含所有的 PTE）和那 4KB 的页目录，都必须被映射到操作系统的虚拟地址空间中。

---

## 模块二：核心概念 —— 什么是页目录自映射？
**1. 自映射的定义**
*   **自映射（Self-Mapping）**：在页目录中，有一条特殊的页目录项（PDE），它指向的**物理地址就是页目录自身的物理地址**。
*   **本质**：通过这条特殊的 PDE，使得 4KB 的页目录表在逻辑上变成了那 4MB 页表空间的一部分（即页目录表是页表的一个子集）。

**2. 自映射的数学意义与比喻 **
*   **地图比喻**：你手持一张北京地图站在北京的某个地方。在地图上**必然有一个点，它在地图上的位置，和它所表示的实际真实地理位置完全重合**。这就是自映射的不动点（Fixed point: $f(x) = x$）。
*   **特性**：“页目录自映射”是虚拟地址空间内部组织的一个特征，**与虚拟地址到物理地址的转换无关**，支持自映射可以为系统节省 4KB 的虚拟地址空间。

## 模块三：自映射的三大核心公式 ⚠️[必考计算，请死记硬背！]
在所有的考试大题中，我们都可以利用“拼接”思想通过位运算（移位与或运算）直接秒杀答案。

**前提条件**：假设将 4MB 的完整页表映射在虚拟地址空间中，其**起始虚拟地址为 `PTbase`**。
*   `PTbase` 必须是 **4MB对齐** 的（即其低22位全为0，例如 `0x8000 0000` 或 `0x7FC0 0000`）。

**🔑 公式 1：页表的起始虚拟地址（已知量）**
> $PTbase = ((PTbase) >> 22) << 22$

**🔑 公式 2：一级页表（页目录表）的起始虚拟地址 `PDbase`**
> $PDbase = PTbase \ | \ (PTbase >> 10)$

**🔑 公式 3：映射自己的那条表项（自映射PDE）的虚拟地址 `PDE_{self-mapping}`**
> $PDE_{self-mapping} = PTbase \ | \ (PTbase >> 10) \ | \ (PTbase >> 20)$

## 模块四：底层推导逻辑（为什么公式长这样？）💡[帮助理解原理]
PPT中给出了两种推导思路（以 `PTbase = 0x7FC00000` 为例）：

**思路一：按 4MB 空间计算**
1.  `0x7FC0 0000` 右移 22 位（`>> 22`），等于 `0x1FF`（十进制511）。说明页表在这个虚拟空间的第 511 个 4MB 块。
2.  因此，页目录就存储在第 511 个页表页中。
3.  每个页表页大小 4KB（$2^{12}$）。第 511 个页表页在 4MB 页表空间内的偏移是：`0x1FF << 12 = 0x1FF000`。
4.  加上基址 `PTbase`，**页目录地址 = `0x7FC0 0000 + 0x1FF000 = 0x7FDF F000`**。

**思路二：按 PTE 个数计算（更直接）**
1.  `0x7FC0 0000` 右移 12 位（`>> 12`），得到 `0x7FC00`。这说明它对应整个地址空间的第 `0x7FC00` 个 4KB 页面。
2.  因此它的映射关系就记录在整个 4MB 页表的第 `0x7FC00` 个页表项（PTE）中。
3.  每个 PTE 占 4 字节。该 PTE 的偏移量 = `0x7FC00 * 4 = 0x1FF000`。
4.  加上基址，得到**页目录地址 = `0x7FDF F000`**。




## 模块四补充：简化计算公式与反向计算 ⚠️[快速秒杀]

### 🔑 简化计算公式（PPT直接给出）

对于 **32位地址字长，2级页表，4KB页面大小**：

给定页表起始地址（虚拟地址，**4MB对齐**）X

```
页目录起始地址 = X + (X >> 10) = X + X / 1024 （先十六进制右移5位，再加上除以4的数）
```

**练习示例**：
- 页表起始地址 `0x80000000`，页目录起始地址 = ?
- 计算：`0x80000000 + 0x00200000 = 0x80200000`

---

### 🔑 反向计算公式（已知 PDbase 求 PTbase）

**核心公式**：
```
PT_base = (PD_base >> 22) << 22
```

> 本质：提取高10位，低22位清零（恢复4MB对齐）

**练习示例**：
- 页目录起始地址 `0xC0300000`，页表起始地址 = ?
- 计算：`(0xC0300000 >> 22) << 22 = 0x300 << 22 = 0xC0000000`

## 模块五：历年真题实战演练 ⚠️[期中期末大题模拟]

结合历年真题，这种题型通常占 5~6 分，完全是套公式的“送分题”或“送命题”。

### 🎯 案例 1：2025年期中考试真题
**题目**：一个32位的虚拟存储系统有两级页表... 如果从 `0x2C00 0000` 开始映射 4MB 的页表，请给出：
1. **一级页表（页目录）的起始虚拟地址**。
2. **一级页表中映射自己的表项的虚拟地址**。

**解题（套公式）：**
*   已知 `PTbase = 0x2C00 0000`
*   `PTbase >> 10 = 0x000B 0000`
*   `PTbase >> 20 = 0x0000 02C0`
*   **第一问 (`PDbase`)**：`0x2C00 0000 | 0x000B 0000 = 0x2C0B 0000`
*   **第二问 (`PDE_{self-mapping}`)**：`0x2C0B 0000 | 0x0000 02C0 = 0x2C0B 02C0`
*(答案完美契合参考答案)*

### 🎯 案例 2：2024年期中考试真题
**题目**：如果从 `0x8C00 0000` 开始映射 4MB 的页表，求同样的两个地址。

**解题（套公式）：**
*   已知 `PTbase = 0x8C00 0000`
*   `PTbase >> 10 = 0x0023 0000`
*   `PTbase >> 20 = 0x0000 08C0`
*   **第一问 (`PDbase`)**：`0x8C00 0000 | 0x0023 0000 = 0x8C23 0000`
*   **第二问 (`PDE_{self-mapping}`)**：`0x8C23 0000 | 0x0000 08C0 = 0x8C23 08C0`

### 🎯 案例 3：2021年期末考试真题变形
**题目**：假设第二级页表的起始逻辑地址（即 `PTbase`）为 `0x1FC0 0000`，求一级页表的起始逻辑地址。
**解题**：
*   `PTbase = 0x1FC0 0000`
*   `PTbase >> 10 = 0x0007 F000`
*   **页目录逻辑地址 (`PDbase`)**：`0x1FC0 0000 | 0x0007 F000 = 0x1FC7 F000`
