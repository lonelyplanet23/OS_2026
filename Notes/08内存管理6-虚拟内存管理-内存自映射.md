# 📚 虚拟内存 - 页目录自映射（考试重点梳理）

---

## 模块一：多级页表与引入背景

### 1.1 一级页表的问题 ★理解
**页表大小与逻辑地址空间大小成正比**。当逻辑地址空间很大时，页表太大，占用大量连续物理内存，分配困难。

**32位系统示例**：
- 页面大小：4KB = 2¹² B
- 页内偏移：12位
- 页数：4GB / 4KB = 1M = 2²⁰ 页
- 每个PTE 4字节 → 页表大小 = 2²⁰ × 4B = **4MB（连续物理内存！）**

**64位系统更夸张**：2⁶⁴ / 2¹² = 2⁵² 页 → 页表大小 = 16 PB！

### 1.2 解决方案
1. **多级页表**：破除页表存储的连续性假设
2. **动态调入页表**：只将当前需用的页表项调入内存，其余需用时再调入

### 1.3 32位二级页表结构 ★★★必考
- 地址结构：**10位（PD索引） + 10位（PT索引） + 12位（页内偏移）**
- 页目录表（PDT）：1K个PDE，占4KB
- 每个PDE对应一个页表页（4KB，含1K个PTE）
- 总大小：4KB + 4MB ≈ 4MB

## 模块二：页表管理的核心问题

### 2.1 关键问题链
| 问题 | 答案 |
|------|------|
| 谁来管理（填写）页表？ | 操作系统OS |
| 填写页表目的？ | 反映内存布局 |
| 如何填写、修改页表？ | 写页表所在内存 |
| 用虚拟地址还是物理地址写？ | **虚拟地址！**（OS运行在保护模式下） |

### 2.2 核心矛盾
页表是用来建立虚拟地址到物理地址映射的，但OS修改页表本身也需要用虚拟地址访问。

→ **页表本身也必须被映射到虚拟地址空间中！**

## 模块三：页目录自映射的核心原理

### 3.1 自映射定义 ★★★
在页目录中，将其中一个**特殊的PDE（页目录项）**指向**页目录自身的物理地址**。

### 3.2 效果
通过这个特殊的PDE，可以用**虚拟地址访问所有页表本身**！

### 3.3 页表空间大小计算（32位）
- 每个PTE 4字节
- 共1M个PTE → 1M × 4B = **4MB**
- 4MB页表分页存储 → 4MB / 4KB = **1024个页表页**
- 每个页表页含 4KB / 4B = 1024个PTE
- 每个页表页对应 1024 × 4KB = **4MB内存**

### 3.4 自映射的作用
1. 操作系统可以**用虚拟地址读写页表**（不需要物理地址）
2. 页表本身也被纳入虚拟地址空间管理
3. 简化页表的创建、修改和销毁操作

### 3.5 典型实现
- 通常将自映射放在虚拟地址空间的最高端（内核空间）
- 例如：在3G以上的内核空间分配4MB作为页表的虚拟地址区域
- OS可以直接用指针访问任何页表项

## 模块四：自映射的地址计算 ★★★★★考试必考

### 4.1 前提条件
- **32位系统，二级页表，4KB页面大小**
- 页表起始虚拟地址 `PTbase` 必须是 **4MB对齐**（低22位全为0）

### 4.2 三大核心公式 ★★★死记硬背！
已知页表起始虚拟地址（4MB对齐）`PTbase = X`

| 计算目标 | 公式 | 说明 |
|---------|------|------|
| **页目录起始虚拟地址 PDbase** | `PDbase = X + (X >> 10)` | 或写成 `X | (X >> 10)`，结果相同 |
| **自映射PDE的虚拟地址** | `PDE_self = PDbase + ((X >> 20) << 2)` | 或写成 `X | (X >> 10) | (X >> 20)` |

> **简化记忆**：页目录地址 = 页表基址 + 页表基址右移10位

### 4.3 反向计算公式
已知 PDbase 求 PTbase：
```
PTbase = (PDbase >> 22) << 22
```
本质：提取高10位，低22位清零（恢复4MB对齐）

---

### 4.4 例题精讲（考试真题模式）

#### 例题1（期中考试真题）
**题目**：从 `0x2C000000` 开始映射4MB的页表，求：
1. 一级页表（页目录）的起始虚拟地址
2. 一级页表中映射自己的表项的虚拟地址

**解答**：
```
已知 PTbase = 0x2C000000

1. PDbase = 0x2C000000 + (0x2C000000 >> 10)
           = 0x2C000000 + 0x000B0000
           = 0x2C0B0000

2. PDE_self = 0x2C0B0000 + (0x2C000000 >> 20) 
            = 0x2C0B02C0
```

#### 例题2（期末考试真题）
**题目**：页目录起始虚拟地址为 `0xC0300000`，求页表起始地址 PTbase。

**解答**：
```
PTbase = (0xC0300000 >> 22) << 22
       = 0x300 << 22
       = 0xC0000000
```

## 模块五：自映射的访问方式（理解）

### 5.1 访问页目录本身
```
PD index = 自映射项的索引
PT index = 自映射项的索引
offset = 要访问的PDE在页目录中的偏移
```
这样组合出的虚拟地址，可以访问到页目录自身的内容。

### 5.2 访问某个页表页
```
PD index = 自映射项的索引
PT index = 目标页表对应的PDE索引
offset = 目标PTE在该页表中的偏移
```

## 模块六：自映射的优势与思考

### 6.1 四大优势
| 优势 | 说明 |
|------|------|
| **简化页表操作** | OS用普通指针就能读写页表，不需要切换到物理地址 |
| **统一内存管理** | 页表本身也作为虚拟内存的一部分被管理 |
| **减少TLB刷新** | 页表区域通常在内核空间，进程切换不需要刷新这部分TLB |
| **易于扩展** | 动态分配和释放页表页变得简单 |

### 6.2 思考题
1. **Q：如果不自映射，OS如何修改页表？**
   A：需要临时建立映射，或者一直用物理地址访问，非常麻烦。

2. **Q：自映射占用多少虚拟地址空间？**
   A：32位系统下占用4MB。

3. **Q：自映射会带来什么问题吗？**
   A：占用4MB虚拟地址空间，对64位系统来说完全不是问题，但32位系统内核空间紧张时需要精打细算。
