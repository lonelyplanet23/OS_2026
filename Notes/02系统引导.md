## 启动的纠结性
+ **矛盾**：必须通过启动程序启动计算机，而启动程序又必须在启动好的计算机上运行（鸡生蛋问题）
+ **核心思想**：**逐步释放系统灵活性**，从最安全、通用、功能最弱的状态逐步设置硬件
+ **英文术语**：Bootstrapping（简称boot），源自"拉鞋带"比喻

---

## Bootloader
+ **定义**：系统加电后运行的第一段小程序，在OS kernel之前执行
+ **作用**：初始化硬件设备、建立内存空间映射图，为调用操作系统内核做准备
+ **两阶段结构**：
  + **Stage1**：依赖CPU体系结构，用汇编实现
    - 初始化硬件：watchdog、中断、时钟、内存
    - 为Stage2准备RAM空间
  + **Stage2**：用C语言实现，可读性和移植性好
    - 初始化本阶段硬件设备
    - 将内核镜像读到RAM，设置启动参数
    - 跳转至内核入口，移交控制权

### 常见Bootloader
| 系统 | Bootloader |
|------|-----------|
| MIPS嵌入式 | U-Boot |
| X86 | LILO / GRUB |

## MIPS架构启动

### MIPS基本地址空间（32位，4G）
+ **kuseg (0-2G)**：用户态唯一可用地址空间，需TLB映射
+ **kseg0 (2G-2.5G)**：内核态，直接映射（最高位清零）到物理内存低512M，经Cache
+ **kseg1 (2.5G-3G)**：内核态，直接映射（高三位清零）到物理内存低512M，**不经Cache**
  - **★ 唯一系统重启时能正常工作的地址空间**
+ **kseg2 (3G-4G)**：内核态，需TLB映射

### MIPS启动地址
+ **启动入口**：`0xBFC00000`（kseg1区域）
+ 对应物理地址：`0x1FC00000`（&0x1FFFFFFF）

### U-Boot启动流程
+ **Stage1（汇编）**：硬件初始化 → 为Stage2初始化RAM → 载入Stage2到RAM → 设置堆栈跳转
+ **Stage2（C代码）**：
  - `board_init_f`：timer_init、env_init、init_baudrate、serial_init、console_init_f、display_banner、init_func_ram
  - `relocate_code`：代码从Flash搬到RAM
  - `board_init_r`：初始化flash、pci、外设 → 进入命令行或启动Linux kernel

## X86架构启动

### 第一步：加载BIOS
+ **BIOS**：Basic Input/Output System，固化在主板ROM芯片的固件
+ **启动地址**：CPU加电后跳转到固定物理地址 **0xFFFF0**（Intel 80386）
+ **核心功能**：
  - POST（Power-On Self Test）：硬件自检，有问题主板发出不同蜂鸣
  - 识别启动设备，按启动顺序加载
  - 提供最底层硬件设置和控制

#### ★ BIOS局限性（考试重点）
| 局限维度 | 具体表现 |
|---------|---------|
| 架构陈旧 | 16位模式，寻址仅1MB，汇编编写 |
| 功能受限 | 仅支持MBR分区（≤2TB），扩展卡驱动空间仅128KB |
| 安全脆弱 | 简易密码保护，存在安全漏洞 |
| 体验不佳 | 纯文字界面，仅键盘操作 |

### UEFI（统一可扩展固件接口）
+ BIOS的替代方案，Intel 2000年提出
+ **优势**：32/64位模式，支持GPT分区（>2TB），模块化C语言设计，网络能力强

---

### 第二步：读取MBR
+ **MBR**：Master Boot Record（主引导记录）
+ **位置**：硬盘第0磁头第0磁道第一个扇区，共 **512字节**
+ **★ 结构（必考）**：
  | 偏移 | 大小 | 内容 |
  |------|------|------|
  | 0x000 | 446B | Bootstrap code area（启动代码） |
  | 0x1BE | 64B | 分区表DPT（4个分区项×16B） |
  | 0x1FE | 2B | **Magic Number: 0xAA55**（标志有效MBR） |

+ **分区表项（16字节）**：
  - 第1字节：引导标志（**80H=活动分区，00H=非活动**）
  - 第5字节：分区类型（**83H=Linux分区，07H=NTFS，0BH=FAT32**）

+ **分区限制**：最多4个主分区，主分区+扩展分区≤4个，仅1个活动分区

---

### 第三步：Boot Loader
+ **LILO**：Linux Loader，配置变更需重写MBR，出错无法引导
+ **★ GRUB**：GRand Unified Bootloader（主流）
  - 有交互式命令界面
  - 支持网络引导
  - 配置错误仅进入命令行，不会系统无法引导

#### GRUB三阶段：
1. **Stage1**：读取MBR（512字节）
2. **Stage1.5**：识别文件系统格式
3. **Stage2**：加载引导菜单，加载内核映像和initrd

+ **加载地址**：BIOS将MBR读入内存 **0x7C00** 处执行

---

### 第四步：加载Linux内核
+ **内核镜像类型**：
  - zImage：<512KB
  - bzImage：>512KB（big zImage）

+ **★ 内核加载流程**：
  1. **bootsect.S**：
     - 自身从0x7C00移至0x90000
     - 加载setup到0x90200
     - 加载内核到0x100000（高端内存）
     - 跳转到setup入口
  2. **setup.S**：
     - 初始化硬件设备，获取内存布局
     - 启动A20地址线，建立IDT/GDT
     - **切换CPU从实模式到保护模式**（CR0.PE=1）
     - 跳转到`startup_32()`
  3. **Head.S（第一个startup_32）**：
     - 初始化段寄存器和堆栈
     - 调用`decompress_kernel()`解压内核（显示"Uncompressing Linux..."）
     - 跳转到解压后的内核位置0x100000
  4. **Head.S（第二个startup_32）**：
     - 初始化最终段寄存器
     - 初始化临时内核页表
     - **跳转到start_kernel()（第一个C函数）**

+ **start_kernel()初始化**：
  - 调度器初始化`sched_init()`
  - 内存初始化：伙伴系统、slab分配器
  - 中断初始化：`trap_init()`、`init_IRQ()`
  - 时间初始化`time_init()`

---

### 第五步～第十步：用户空间初始化（★ 顺序记忆）
5. **init进程启动**：内核加载后第一个运行程序`/sbin/init`，读取`/etc/inittab`设定运行级别
6. **执行rc.sysinit**：设定PATH、网络配置、启动swap、挂载/proc
7. **启动内核模块**：依据`/etc/modules.conf`加载
8. **执行运行级别脚本**：根据runlevel执行rc0.d~rc6.d中脚本
9. **执行rc.local**：用户自定义启动脚本（个性化设置位置）
10. **执行/bin/login**：进入登录状态

#### Linux运行级别（★ 必考）
| 级别 | 含义 |
|------|------|
| 0 | 关机 |
| 1 | 单用户模式 |
| 2 | 无网络多用户 |
| 3 | 有网络多用户（文本界面） |
| 4 | 保留未用 |
| 5 | 有网络+X-Window（图形界面） |
| 6 | 重启 |

## 思考与扩展
+ **为什么加载到0x7C00？**：历史遗留，BIOS中断int19规定
+ **为什么是16位？**：DOS是16位，X86向后兼容
+ **多核启动区别**：主核（BSP）完成启动，其他核（AP）稍后由主核唤醒
