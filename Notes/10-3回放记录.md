## 模块五：实时调度与多核调度 (高级特性)
**实时系统**
+ 计算机必须在**确定的时间范围**内做出反应。
+ 硬实时系统/软实时系统
+ 
**1. 实时调度 (RMS vs EDF)**
**重点：要感知优先级，本质是调整不同任务的offset，由于实时，因此可调度是有条件的**
【用latex改写】
事先确定以下参数，假设一任务集 S = { t₁, t₂, t₃, …, tₙ }，
- 每个任务出现的周期分别是 T₁, T₂, …, Tₙ
- 执行时间为 C₁, C₂, …, Cₙ (Ci < Ti)
- 截至周期(deadline)为 D₁, D₂, …, Dₙ，令 Dᵢ = Tᵢ
- CPU利用率：U = Σ (Cᵢ / Tᵢ)
- 任务独立、上下文切换可忽略

+   **静态表调度**：通过对所有周期性任务的分析预测（到达时间、运行时间、结束时间、任务间的优先关系），事先确定一个固定的**调度方案**（本质上是确认各个任务的offset）。 
    +   无需计算开销小、没有灵活性。

*   **单调速率调度 (RMS) Rate Monotonic Scheduling**：
    *   **静态优先级**
    *   充分条件：Σ (Cᵢ / Tᵢ) ≤ n(ⁿ√2 - 1)
    *   **周期越短，优先级越高**。
    +   **抢占式**
    *   优点：是**单核下静态调度的最优算法**，但CPU利用率有理论上限（约69%）。
    *   适用：硬实时系统（如航天控制），确定性
    *   缺点：有的长周期任务可能错过ddl
  ![18](assets/image-18.png)

*   **最早截止期优先 (EDF) Earliest Deadline First**
    *   条件：有可用的CPU - iff：Σ (Cᵢ / Tᵢ) ≤ 1
    *   **动态优先级**
    *   **绝对截止时间越近，优先级越高**。理论上可以达到100%的CPU利用率。
    *   **抢占式：如果新任务【某一任务下一周期开始】离截止时间更近，则抢占**
    *   缺点：实现相对复杂（优先队列、堆），动态处理；过载行为难预测，低优先级因截止时间早而抢占关键任务

**2. 优先级倒置 (Priority Inversion) ⚠️[高频名词解释]**
*   **场景**：高优先级任务A想获取资源锁，被低优先级任务C占用（A阻塞）；此时中优先级任务B抢占了C的CPU，导致A被无限期延迟（如著名的NASA火星车重启故障）。
*   **解决方案**：
    *   **优先级置顶 (Priority Ceiling)**：占用资源的进程，优先级自动提至所有可能申请该资源进程的最高优先级。
    *   **优先级继承 (Priority Inheritance)**：低优先级进程C临时“继承”被它阻塞的高优先级进程A的优先级，直到释放资源。

**3. 多处理机调度 (Multi-Processor)**
**注重整体运行效率，多处理机访问数据互斥**
**AMP 非对称式多处理：多个处理器地位不同**
+ 主从处理机（例子：神威太湖之光）
**SMP 对称式多处理系统：~相同**
+ **自调度：整个系统有一个公共的就绪队列，每个处理机从中选取一个进程运行**
*   **Cache 亲和性 (Cache Affinity)**：尽量让进程保持在同一个CPU上运行，以利用L1/L2 Cache中的遗留状态，避免Cache失效带来的巨大惩罚。
*   **工作窃取 (Work Stealing)**：多队列调度(MQMS)中，为了解决某些CPU过载而其他CPU闲置的问题，空闲CPU的队列会去别的繁忙队列中“偷”任务过来执行。
