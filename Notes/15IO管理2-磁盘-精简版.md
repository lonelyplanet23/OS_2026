### 📚 一、磁盘基本概念：如何定位一个磁盘块？

#### 1. 磁盘物理结构

磁盘由多个盘片组成，每个盘片通常有两个面，每个面都有一个磁头（head）。磁头沿半径方向移动，盘片高速旋转完成读写。

**每个磁盘有两个面，每个面都有一个磁头(head)**

![alt text](assets/image-32.png)

| 概念 | 含义 | 定位作用 |
| --- | --- | --- |
| 扇区（sector） | 盘片上的扇形区域，是读写基本区域之一 | 指定磁道上的具体扇区 |
| 磁道（track） | 盘面上不同半径的同心圆 | 指定磁头移动到哪个半径 |
| 柱面（cylinder） | 不同盘片相同半径的磁道组成的圆柱面 | 多磁头在同一半径对应的一组磁道 |

【磁道+扇区 定位块】

【磁头组：这些磁头只能读不同盘片的同一个柱面】

磁盘定位可理解为三维定位：柱面 → 磁头 → 扇区。

```text
磁盘块定位
  │
  ├─ 柱面 Cylinder：磁头臂移动到哪个半径
  ├─ 磁头 Head：选择哪个盘面
  └─ 扇区 Sector：等待旋转到哪个扇形区域
```

#### 2. CHS 与 LBA

磁盘地址可表示为：

$$
\text{磁盘地址} = (\text{柱面}, \text{磁头}, \text{扇区})
$$

现代磁盘更多抽象为一维逻辑块数组，每个逻辑块有 LBA（Logic Block Address）。OS 使用逻辑块号，磁盘控制器负责映射到底层物理位置。

| 符号 | 含义 |
| --- | --- |
| C | 每盘面柱面数 |
| H | 磁头数，即盘面数 |
| S | 每磁道扇区数 |

若已知 `(c, h, s)`，逻辑块号可表示为【无需掌握，了解即可】：

$$
\text{块号}=c\times(H\times S)+h\times S+s
$$

#### 3. 磁盘缺陷与扇区替换

实际磁盘可能存在缺陷扇区，磁盘内部会维护缺陷列表，用保留扇区替代坏扇区。

#### 4. MBR：磁盘从哪里启动？

MBR（Master Boot Record）位于硬盘 **0 柱面、0 磁头、1 扇区**，大小 **512 字节**，是传统 BIOS 启动入口，用于把控制权交给分区表中标记为可启动的 OS 分区。

```text
BIOS 启动流程（传统 MBR）
  │
  ├─ 读取硬盘 0 柱面 0 磁头 1 扇区
  ├─ 检查 MBR 末尾魔数 55AA
  ├─ 执行前 446 字节启动代码
  ├─ 查找分区表中的 active 主分区
  └─ 跳转到对应分区引导扇区 DBR，继续启动 OS
```

| 区域 | 大小 | 作用 |
| --- | ---: | --- |
| 启动代码及数据 | 446 字节 | 存放启动程序，继续引导系统 |
| 分区表 DPT | 64 字节 | 4 个分区项，每项 16 字节 |
| 魔数 Magic Number | 2 字节 | BIOS 判断有效引导扇区 |

💡 **考试常考点：** MBR 的位置、大小、结构与分区数量限制。

#### 5. MBR 分区限制

MBR 中 DPT 只有 64B，每个分区项 16B，因此最多 4 个分区项：

$$
\frac{64\text{B}}{16\text{B}}=4
$$

| 分区类型 | 规则 |
| --- | --- |
| 主分区 | 至少 1 个，最多 4 个 |
| 扩展分区 | 可以没有，最多 1 个；本身不能直接使用 |
| 逻辑分区 | 位于扩展分区内部，可以多个 |
| 主分区 + 扩展分区 | 总数不能超过 4 个 |
| active | 主分区中只能有一个被标记为 active |

```text
MBR 分区表（最多 4 项）
  ├─ 方案 A：主分区 + 主分区 + 主分区 + 主分区
  └─ 方案 B：主分区 + 主分区 + 主分区 + 扩展分区
                                      ├─ 逻辑分区 1
                                      ├─ 逻辑分区 2
                                      └─ 逻辑分区 ...
```

#### 6. DBR 与启动相关概念

DBR（DOS Boot Record / Partition Boot Sector）位于某个分区起始位置。MBR 找到并跳转到活动分区，DBR 再加载该分区的 OS 引导程序。

| 概念 | 位置/作用 | 记忆关键词 |
| --- | --- | --- |
| MBR | 整块硬盘第一个引导扇区 | 全盘入口、含分区表 |
| DPT | MBR 内部分区表 | 4 个分区项 |
| DBR | 分区起始处引导扇区 | 分区入口、继续引导 OS |
| active | 可启动主分区标记 | 只能一个主分区激活 |

【考试重点】若题目问 MBR 的本质，应答：MBR 是硬盘 0 柱面、0 磁头、1 扇区上的 512B 主引导记录，包含启动代码、4 项分区表和结束魔数，用于传统 BIOS 启动与分区定位。

### 🏗️ 二、磁盘访问时间：为什么磁盘慢？

#### 1. 三个时间组成

磁盘读写需先寻道，再等待目标扇区旋转到磁头下，最后传输数据。

| 时间 | 含义 | 主要影响因素 |
| --- | --- | --- |
| 寻道时间 `Ts` | 磁头移动到目标磁道所需时间 | 移动距离、启动时间 |
| 旋转延迟 `Tr` | 等待目标扇区旋转到磁头下方 | 磁盘转速 |
| 传输时间 `Tt` | 数据在磁盘与内存/控制器间传输 | 数据量、转速、磁道容量 |

#### 2. 寻道时间

启动时间 `s`，移动 `n` 个磁道：

$$
T_s=m\times n+s
$$

`m` 为移动一个磁道的时间常数，`n` 为移动磁道数，`s` 为启动磁臂时间。

#### 3. 旋转延迟时间

平均旋转延迟为一圈时间的一半：

$$
T_r=\frac{1}{2r}
$$

若题目给 RPM（转/分钟），先换算为：

$$
r=\frac{RPM}{60}
$$

| 转速 RPM | 平均旋转延迟 |
| ---: | ---: |
| 4200 | 7.14 ms |
| 5400 | 5.56 ms |
| 7200 | 4.17 ms |
| 10000 | 3 ms |
| 15000 | 2 ms |

#### 4. 传输时间与总访问时间

读写字节数 `b`，旋转速度 `r`，每磁道字节数 `N`：

$$
T_t=\frac{b}{rN}
$$

总访问时间：

$$
T_a=T_s+\frac{1}{2r}+\frac{b}{rN}
$$

```text
一次磁盘 I/O
  ├─ 1. 寻道：磁头移动到目标磁道
  ├─ 2. 旋转等待：目标扇区转到磁头下方
  └─ 3. 数据传输：读/写目标数据
```

💡 **判断主要矛盾，具体情况具体分析。** 小数据块读写中，寻道与旋转延迟通常占主要部分；数据量很大时，传输时间更明显。

### 🚀 三、磁盘调度算法：如何减少寻道距离？

#### 1. 为什么需要磁盘调度？

当 I/O 请求到达速度高于磁盘服务速度时，请求排队。OS 可重新排列请求顺序，减少磁头移动距离，提高吞吐量。

```text
多个 I/O 请求到达
        │
        ▼
请求队列等待磁盘服务
        │
        ▼
OS 选择调度算法
        │
        ▼
重排访问顺序，减少寻道时间
```

#### 2. FCFS：先来先服务

FCFS 按请求到达顺序服务。

![1](assets/image-33.png)

| 方面 | 内容 |
| --- | --- |
| 算法思想 | 按访问请求到达先后次序服务 |
| 优点 | 简单、公平、**不会饥饿** |
| 缺点 | **平均寻道距离可能较大**，磁头可能反复大范围移动 |
| 适用场景 | 磁盘 I/O 较少、请求不密集 |

#### 3. SSTF：最短寻道时间优先

SSTF 总是优先满足距离当前磁头最近的请求，属于贪心策略。

![2](assets/image-34.png)

| 方面 | 内容 |
| --- | --- |
| 算法思想 | 优先选择距当前磁头最近的请求 |
| 优点 | 通常比 FCFS 平均寻道时间短 |
| 缺点 | 可能产生“饥饿”，远处请求长期等待 |
| 适用场景 | 追求平均服务时间、请求分布相对稳定 |

#### 4. SCAN：扫描算法 / 电梯调度

SCAN 像电梯：磁头沿一个方向移动，沿途服务请求；**到达该方向边界后，再反向服务。**（检查）

![35](assets/image-35.png)

| 方面 | 内容 |
| --- | --- |
| 算法思想 | 按一个方向移动，沿途服务，到端点后反向 |
| 优点 | 同时考虑距离和方向，避免 SSTF 严重饥饿 |
| 缺点 | 两侧磁道访问频率低于中间磁道 |
| 适用场景 | 请求较多，需要兼顾公平与性能 |

```text
SCAN（向大柱面方向）
当前位置
  ├─ 向大柱面移动，服务沿途请求
  ├─ 到达磁盘边界/方向终点
  └─ 反向移动，服务反方向请求
```

#### 5. LOOK：对 SCAN 的改进

LOOK 不一定扫描到物理端点，而是“查看”该方向上是否还有请求；若没有，就立即改变方向。

- LOOK 查看扫描算法：“**判断该方向上是否还有访问请求**，如果有则继续扫描；否则改变移动方向”
- 只需到达该方向最远请求即可返回，不必到达磁盘端点。

#### 6. C-SCAN：循环扫描算法

C-SCAN 始终按同一方向服务；到达末端后快速返回 0 号柱面，返回过程中不服务请求。

| 方面 | 内容 |
| --- | --- |
| 算法思想 | 单向扫描服务，到末端后快速回起点 |
| 优点 | 对各位置请求更公平，等待时间更均匀 |
| 缺点 | 返回过程中不服务请求，会产生额外移动 |
| 适用场景 | 希望等待时间更均匀 |

**辨析**：C-SCAN 解决的是 SCAN 中**请求等待时间不均衡**的问题，而非饥饿问题。SCAN 本身一定能在一个周期内满足请求；SSTF 才可能因近处请求源源不断而饥饿。

#### 7. C-LOOK、N-Step-SCAN 与 FSCAN（了解）

| 算法 | 思想 | 解决的问题 |
| --- | --- | --- |
| C-LOOK | C-SCAN 的 LOOK 版本，到最远请求后返回最小请求端 | 减少无请求端点移动 |
| N-Step-SCAN | 队列分为长度 N 的子队列，队列间 FCFS，队列内 SCAN | 缓解磁臂黏着 |
| FSCAN | 当前旧请求队列和新请求队列交替扫描 | 新请求不持续插队影响旧请求 |

**磁臂黏着**：当前磁头附近不断产生请求，导致远处请求长期无法满足。

#### 8. 调度算法对比

| 算法 | 优点 | 缺点 | 记忆关键词 |
| --- | --- | --- | --- |
| FCFS | 公平、简单 | 平均寻道距离大 | 按到达顺序 |
| SSTF | 平均服务时间好 | 可能饥饿 | 离当前最近 |
| SCAN | 性能较好，可避免严重饥饿 | 两端请求相对不利 | 电梯，到边界再反向 |
| C-SCAN | 等待时间更均匀 | 返回过程不服务 | 单向扫描，快速回 0 |
| LOOK | 少走无请求端点 | 题目需明确是否默认 LOOK | 到最远请求即返回 |
| C-LOOK | 少走无请求端点 | 返回过程仍不服务 | 循环 LOOK |

【题型：给定柱面访问请求序列，用相应算法计算总访问道数，主要是SCAN FCFS SSTF三个】

### 🔄 四、磁盘空间管理：如何记录空闲块？（略）

| 方法 | 基本思想 | 特点 |
| --- | --- | --- |
| 位图 | 每个物理块对应一位，用 0/1 表示分配状态 | 查找连续空闲块方便，但位图占空间 |
| 空闲表法 | 记录每段连续空闲区起始块号和块数 | 适合连续分配，表项动态变化 |
| 空闲链表法 | 将所有空闲块链接起来 | 不要求连续表，但链可能很长 |
| 成组链接法 | 空闲块分组，每组记录下一组信息 | 节省空间，分配/回收较高效，采用栈思想 |

### ⚖️ 五、非机械式存储：Flash Disk 与 SSD

#### 1. Flash Disk 特点

Flash Disk 无高速旋转盘片和移动磁头，访问延迟远小于机械硬盘。

| 特点 | 说明 |
| --- | --- |
| 低功耗 | 没有复杂机械部件 |
| 数据访问速度高 | 没有寻道和旋转延迟 |
| 读写周期短 | NAND Flash 可达微秒级，硬盘常为毫秒级 |
| 抗震性好 | 无高速机械结构 |

存储技术：NOR（宽数据线，一次性寻址）、NAND（块号、块内页号、页内字节号）。

#### 2. NOR 与 NAND

| 对比项 | NOR Flash | NAND Flash |
| --- | --- | --- |
| 随机读取 | 较快 | 相对较慢 |
| 连续大数据传输 | 与 NAND 差异较小 | 适合大容量连续数据 |
| 成本 | 每 MB 成本高 | 成本低、容量大 |
| 典型用途 | 代码存储、直接寻址 | SSD、U 盘、存储卡 |

#### 3. NAND 写入与擦除

**NAND**

- 写开销远大于读：必须**成块读写**，写前通常要先擦除块；
- 寿命问题类比：纸变薄后用橡皮擦容易擦破。

![alt text](assets/image-36.png)

课程规律：writes 10x reads, erasure 10x writes，即写比读慢，擦除又比写更慢。

#### 4. FTL 与磨损均衡

解决：**FTL** 技术，决定擦哪块、映射哪块，使不同物理块均匀被擦写，并支持恢复。

FTL（Flash Translation Layer）维护逻辑块到物理闪存页/块的映射，并配合磨损均衡（Wear Leveling）、垃圾回收、坏块管理，延长寿命。

```text
操作系统看到：逻辑块 LBA
        │
        ▼
FTL 映射层
  ├─ 逻辑页 → 物理页
  ├─ 垃圾回收
  ├─ 坏块管理
  └─ 磨损均衡
        │
        ▼
真实 NAND Flash 块/页
```

#### 5. Flash Disk 是否适用传统磁盘调度？

机械硬盘调度核心是减少寻道和旋转延迟；Flash Disk 没有磁头与盘片，因此 FCFS、SSTF、SCAN、C-SCAN 等按柱面距离优化的算法不再是主要矛盾。

| 存储介质 | 主要延迟来源 | 调度重点 |
| --- | --- | --- |
| HDD | 寻道、旋转延迟、传输时间 | 减少磁头移动，优化访问顺序 |
| SSD / Flash | 控制器并行度、擦写放大、垃圾回收、磨损均衡 | 合并请求、减少随机写、发挥通道并行、延长寿命 |

💡 **答题关键词：** 对 Flash Disk，传统磁盘调度算法“不再直接适用/意义大幅下降”，因为没有机械寻道和旋转延迟；但 I/O 合并、队列调度、负载均衡、减少写放大仍重要。

### 💡 六、RAID：用多块磁盘提高性能与可靠性

#### 1. RAID 基本概念

RAID：独立硬盘冗余阵列。基本思想是把多块磁盘组合成一个逻辑阵列，以提升性能、容量或可靠性。

| 技术 | 含义 | 主要效果 |
| --- | --- | --- |
| 分条（striping） | 数据分条带分布到多个磁盘 | 提高并行读写能力 |
| 镜像（mirroring） | 同一数据保存到多个磁盘 | 提高可靠性和读性能 |
| 校验（parity） | 用 XOR 等保存冗余信息 | 故障时重建数据 |

```text
RAID 的核心目标
  ├─ 性能：多盘并行读写
  ├─ 可靠性：冗余、镜像、校验
  └─ 容量：多盘组合成逻辑存储单元
```

#### 2. RAID 0：条带化

| 项目 | 内容 |
| --- | --- |
| 技术 | 条带化 |
| 优点 | 读写性能高，容量利用率高 |
| 缺点 | **没有冗余**，任一磁盘故障都可能丢数据 |
| 适用 | 高性能、可接受数据丢失风险的临时数据 |
| 示意 | ![41](assets/image-41.png) |

#### 3. RAID 1：镜像

RAID 1 将**同一份数据写入两个或多个磁盘**。

| 项目 | 内容 |
| --- | --- |
| 技术 | 镜像 |
| 优点 | 可靠性高，**读操作可并行** |
| 缺点 | **容量利用率低**，通常为 50% |
| 适用 | 系统盘、关键数据 |
| 示意 | ![42](assets/image-42.png) |

#### 4. RAID 0+1 与 RAID 1+0

| 级别 | 结构 | 可靠性理解 |
| --- | --- | --- |
| RAID 0+1 | 先条带，再对整个条带集合镜像 | 某条带组失效后冗余能力下降明显 |
| RAID 1+0（RAID 10） | 先镜像，再在镜像组之间条带 | 通常比 RAID 0+1 更可靠，也更常用 |

![alt text](assets/13683fba318dfd1a8004630ced1422e5.jpg)

#### 5. RAID 2：海明码校验

RAID 2 使用位级条带化和海明码进行错误检测与恢复。

| 项目 | 内容 |
| --- | --- |
| 技术 | 海明码校验，位级条带化 |
| 优点 | 可自动检测和纠正错误 |
| 缺点 | 校验盘多，实现复杂，现代系统少用 |

#### 6. RAID 3：专用奇偶校验盘，字节/位级条带化

RAID 3 使用数据位/字节交叉，一个冗余校验盘保存奇偶校验。任一盘坏可用 XOR 恢复：

```text
xi = 校验码 XOR 其他盘数据
```

![37](assets/image-37.png)

| 项目 | 内容 |
| --- | --- |
| 优点 | 奇偶校验，可恢复一块盘故障 |
| 缺点 | **字节**条带化，读写需访问所有盘，恢复慢，有水桶效应 |

#### 7. RAID 4：专用校验盘，块级条带化

![38](assets/image-38.png)

| 项目 | 内容 |
| --- | --- |
| 优点 | 块级条带化，支持独立小读，随机读快 |
| 缺点 | 所有写都要更新专用校验盘，校验盘成为瓶颈，**随机写慢** |

#### 8. RAID 5：分布式校验，块级条带化

RAID 5 将校验块分布到所有磁盘，避免 RAID 4 单校验盘瓶颈。

![39](assets/image-39.png)

| 项目 | 内容 |
| --- | --- |
| 优点 | 容量利用率较高，可容忍一块磁盘故障，读性能好 |
| 缺点 | 小写需读旧数据/旧校验并写新数据/新校验，即“写惩罚” |
| 容量 | N 块盘中相当于 1 块盘用于校验 |

RAID 5 小写的四个实际操作：

```text
读旧数据 + 读旧校验 + 写新数据 + 写新校验
```

两块盘坏掉会导致整个 RAID 数据失效。

#### 9. RAID 6：双重分布式校验

RAID 6 类似 RAID 5，但增加两层独立校验信息，可容忍两块磁盘同时故障。

![40](assets/image-40.png)

| 项目 | 内容 |
| --- | --- |
| 优点 | 可容忍两块磁盘同时故障 |
| 缺点 | 写开销更大，容量相当于损失 2 块盘 |
| 适用 | 大容量阵列、重建时间长、故障概率更高的场景 |

#### 10. RAID 级别对比

重点关注磁盘数量与最少磁盘数量。

| 级别 | 说明 | 特点 | 磁盘数量 | 最少磁盘数量 |
| --- | --- | --- | --- | --- |
| 0 | 条带化 | 速度加倍，可靠性减半，容量不变 | n | 2 |
| 1 | 镜像 | 速度不变，可靠性加倍，容量减半 | n × 2 | 2 |
| 0+1 | 组内条带化、组间镜像 | 速度加倍，可靠性加倍，容量减半 | n × 2 | 4 |
| 1+0 | 组内镜像、组间条带化 | 速度加倍，可靠性加倍，容量减半 | 2 × n | 4 |
| 2 | 海明码校验 | 校验独立存储，数据位交叉，自动纠正 1 个盘错 | n + log(n) | 7 |
| 3 | 奇偶校验 | 校验独立存储，数据位交叉，容 1 块盘错 | n + 1 | 3 |
| 4 | 奇偶校验 | 校验独立存储，数据块交叉，容 1 块盘错 | n + 1 | 3 |
| 5 | 奇偶校验 | 校验交叉存储，数据块交叉，容 1 块盘错 | n + 1 | 3 |
| 6 | 奇偶校验 | 校验交叉存储，数据块交叉，容 2 块盘错 | n + 2 | 4 |

#### 11. 平均损坏时间 MTTF

平均损坏时间 **MTTF（Mean Time To Failure）** 表示设备从开始工作到发生故障的平均时间。考试常把故障看作近似**独立事件**，用单位时间故障率计算：

```text
单块磁盘平均损坏时间 = MTTF
单块磁盘故障率 λ = 1 / MTTF
系统平均损坏时间 = 1 / 系统故障率
```

若 n 块相同磁盘，每块 MTTF 为 T，任一块坏系统就坏：

```text
RAID 0：任意一块坏就坏，可靠性下降  --- 串联，事件的交
RAID 1：需要所有镜像盘都坏才坏，可靠性提高 --- 并联，独立事件同时发生
```

RAID 0：

$$
\lambda_{RAID0}=n\lambda=\frac{n}{T}
$$

$$
MTTF_{RAID0}=\frac{1}{\lambda_{RAID0}}=\frac{T}{n}
$$

RAID 1（2 块镜像盘，平均修复时间 $MTTR=R$）：

$$
MTTF_{RAID1}\approx\frac{T^2}{2R}
$$

#### 12. RAID 概念小结

| 概念 | 优点 | 缺点 |
| --- | --- | --- |
| 条带化 | 并行存取，性能好，负载均衡 | 可靠性下降，不同 I/O 可能排队 |
| 镜像 | 可靠性高，读性能可能提升 | 存储开销大 |
| 校验 | 提高可靠性，可恢复 | 需要计算和存储开销，写可能变慢 |

💡 **期末选择题重点：** RAID 5 可容忍 1 块盘故障；RAID 6 可容忍 2 块盘故障。

### 🔧 七、提高磁盘 I/O 速度/性能（了解）

#### 1. 主要途径

课程 PPT 总结：选择性能好的磁盘、并行化、采用适当调度算法、设置磁盘高速缓冲区。【总结为：以上所学主要内容 + 提高磁盘速度 + 下面的内容】

#### 2. 磁盘高速缓存

磁盘高速缓存减少真实磁盘访问次数，将近期使用或即将使用的数据暂存在内存中。

| 缓存形式 | 特点 |
| --- | --- |
| 独立缓存 | 大小固定，专门作为磁盘缓存 |
| 虚拟内存作为缓存 | 大小弹性，可随负载变化 |

| 数据交付方式 | 含义 | 特点 |
| --- | --- | --- |
| 直接交付 | 将缓存数据复制给请求进程 | 有 copy 开销 |
| 指针交付 | 直接交付缓存块指针 | 减少复制，内存管理复杂 |

#### 3. 周期性写回

周期性写回是系统定期将 disk cache 中被修改过的内容写回磁盘。

```text
写请求到达
  │
  ▼
先写入磁盘缓存
  ├─ 稍后周期性写回磁盘
  └─ 若系统崩溃，可能丢失尚未写回的数据
```

#### 4. 优化数据布局、提前读与延迟写

| 方法 | 作用 / 思想 | 风险或代价 |
| --- | --- | --- |
| 连续摆放 | 顺序访问性能高，减少寻道 | 不利于动态增长 |
| 优化索引节点分布 | 减少 inode 与数据块距离 | 需布局管理 |
| 提前读 | 顺序访问时预取下一块 | 预测错误浪费缓存和 I/O |
| 延迟写 | 写请求暂挂队列，稍后合并写回 | 崩溃时可能丢数据 |

延迟写核心：将本应立即写回的数据挂到空闲缓冲区队列末尾，直到移到链头时才写回磁盘，再作为空闲区分配。

#### 5. 虚拟盘（RAM Disk）

| 对比项 | Virtual disk | Disk cache |
| --- | --- | --- |
| 内容控制者 | 用户完全控制 | 操作系统控制 |
| 主要目的 | 用内存模拟高速磁盘 | 缓存磁盘数据，提高访问速度 |
| 数据可靠性 | 掉电易丢，需额外持久化 | 依赖缓存写回策略 |

### 📝 八、考试指南：磁盘管理常见题型

#### 题型 1：磁盘调度算法计算总访问道数

【题型：给定柱面访问请求序列，用相应算法计算总访问道数，主要是SCAN FCFS SSTF三个】

```text
第一步：写出请求序列、初始磁头位置、初始移动方向
        ▼
第二步：按算法规则排出实际访问顺序
        ▼
第三步：逐段计算柱面移动距离
        ▼
第四步：求总柱面数
        ▼
第五步：若给出每柱面移动时间，则乘以单位时间
```

##### 例题：SCAN 电梯调度

请求序列：10，22，20，2，40，6，38；初始磁头在 20，向大柱面方向移动，每移动 1 柱面 4ms，磁盘柱面 0-50。SCAN 每次到边界再反向。

初始位置 20，向大方向：

```text
小于 20：10，2，6
等于 20：20
大于 20：22，40，38
```

向大方向访问：20 → 22 → 38 → 40，继续到边界 50，再反向访问 10 → 6 → 2。

```text
实际访问请求柱面顺序：20 → 22 → 38 → 40 → 10 → 6 → 2
```

注意：50 是边界，不是原始请求；访问序列不写 50，但计算移动距离要算到 50。

| 阶段 | 移动 | 柱面数 |
| --- | --- | ---: |
| 初始到 22 | 20 → 22 | 2 |
| 到 38 | 22 → 38 | 16 |
| 到 40 | 38 → 40 | 2 |
| 扫描到边界 | 40 → 50 | 10 |
| 反向到 10 | 50 → 10 | 40 |
| 到 6 | 10 → 6 | 4 |
| 到 2 | 6 → 2 | 4 |
| 合计 |  | 78 |

$$
78\times4\text{ms}=312\text{ms}
$$

💡 **期末重点提醒：** 2019-2020 与 2024 期末都出现该类 SCAN 题。只写答案不给分时，必须写访问时刻、磁头位置、请求队列或清晰过程。

#### 题型 2：有新请求动态到达的 SCAN

若 `t+31ms`、`t+70ms`、`t+91ms` 又产生新请求，注意：

1. 把时间换算为磁头移动柱面数；
2. 判断新请求到达时磁头位置；
3. 新请求若在当前方向前方，可能本轮服务；若已错过，通常等反向或下一轮；
4. 每一步维护请求队列，不能只按最终集合排序。

```text
动态请求题
  ├─ 先按原始队列推进时间线
  ├─ 到达新请求时插入队列
  ├─ 判断新请求是否在当前方向上尚未经过
  └─ 继续按 SCAN 规则服务
```

#### 题型 3：磁盘访问时间计算

```text
第一步：寻道时间 Ts 直接取题目给定平均值
第二步：RPM 转 rps：r = RPM / 60
第三步：平均旋转延迟 = 1 / (2r)
第四步：传输时间 = b / (rN)
第五步：总时间 = Ts + Tr + Tt
第六步：若文件占多个块，按题目假设判断是否每块都需要一次完整访问
```

例如转速 7500 rpm：

$$
r=\frac{7500}{60}=125\ \text{转/秒}
$$

$$
T_r=\frac{1}{2\times125}=0.004\text{s}=4\text{ms}
$$

💡 **易错点：** RPM 是每分钟转数，公式中的 `r` 常按每秒转数使用，必须先除以 60。

#### 题型 5：RAID 概念判断

| 判断点 | 正确结论 |
| --- | --- |
| RAID 0 是否有冗余 | 没有冗余，任一盘坏都可能丢数据 |
| RAID 1 容量利用率 | 通常约 50% |
| RAID 3 是否可容忍一块盘故障 | 可以 |
| RAID 4 的瓶颈 | 专用校验盘成为写瓶颈 |
| RAID 5 为什么分布式校验 | 避免 RAID 4 单校验盘瓶颈 |
| RAID 6 与 RAID 5 容错能力是否相同 | 不同，RAID 6 可容忍两块盘故障 |

#### 题型 6：Flash Disk 与传统磁盘调度

```text
1. 传统磁盘调度算法主要为减少寻道时间和旋转延迟；
2. Flash Disk 没有磁头和旋转盘片，不存在机械寻道与旋转等待；
3. 所以 SCAN/SSTF 这类按柱面距离优化的算法意义明显下降；
4. 但仍需要 I/O 合并、队列管理、减少写放大、磨损均衡等优化。
```

### ✅ 本章速记总结

1. 磁盘定位依赖柱面、磁头、扇区；现代系统更多使用 LBA。
2. MBR 位于 0 柱面 0 磁头 1 扇区，512B，含启动代码、4 项分区表、魔数。
3. 磁盘访问时间由寻道、旋转延迟、传输组成：$T_a=T_s+\frac{1}{2r}+\frac{b}{rN}$。
4. 磁盘调度核心是减少寻道：FCFS 公平，SSTF 可能饥饿，SCAN 像电梯，C-SCAN 等待更均匀。
5. LOOK/C-LOOK 不一定走到磁盘端点，只到最远请求处返回。
6. Flash Disk 没有机械寻道和旋转延迟，传统柱面调度意义下降；重点转向写放大、磨损均衡、FTL 与并行度。
7. RAID 三大技术是条带化、镜像、校验；RAID 5 容 1 块盘故障，RAID 6 容 2 块盘故障。
8. 提高磁盘 I/O 速度的方法包括缓存、优化布局、提前读、延迟写、并行化和合适调度算法。
9. 期末高频题：SCAN 访问序列与寻道总时间、磁盘访问时间公式、RAID 概念判断。
