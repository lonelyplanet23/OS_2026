

## 1. 长程序中的“闪电移动”

在长长的源代码中，不要再狂按方向键。以下命令能让你瞬间到达目的地。

| 命令         | 操作结果                             | 实战场景                                                     |
| ------------ | ------------------------------------ | ------------------------------------------------------------ |
| `gg`         | 跳转到文件**第一行**                 | 回到文件头部查看宏定义或包含的库。                           |
| `G`          | 跳转到文件**最后一行**               | 快速前往文件末尾。                                           |
| `nG` 或 `:n` | 跳转到文件的**第 n 行**              | 编译器报错“第 128 行出错”，输入 `128G` 秒达。                |
| `Ctrl + d`   | 向下翻半页 (Down)                    | 快速且平滑地向下浏览阅读代码。                               |
| `Ctrl + u`   | 向上翻半页 (Up)                      | 快速且平滑地向上回溯代码逻辑。                               |
| `%`          | 在匹配的括号 `{}`, `()`, `[]` 间跳转 | **排错神器**：检查 if/while 语句的长代码块括号有没有正确闭合。 |

## 2. 搜索与精准定位

不要手动找变量，让 Vim 替你找。

| 命令      | 操作结果                                  | 高效技巧                                                     |
| --------- | ----------------------------------------- | ------------------------------------------------------------ |
| `/关键字` | 向下搜索（按 `n` 找下一个，`N` 找上一个） | 查找特定的函数名或报错信息。                                 |
| `?关键字` | 向上搜索（按 `n` 找下一个，`N` 找上一个） | 从当前位置往回查找。                                         |
| `*`       | **精准搜索光标所在的单词（向下）**        | 光标停在某个复杂变量名上，按 `*` 直接高亮并跳到它下次出现的位置，免去手动输入的麻烦。 |
| `#`       | **精准搜索光标所在的单词（向上）**        | 同上，方向相反。                                             |

## 3. 代码缩进与排版

代码格式乱了不仅影响阅读，还容易看错作用域。Vim 的自动排版非常聪明。

| 命令                 | 操作结果                   | 说明                                                         |
| -------------------- | -------------------------- | ------------------------------------------------------------ |
| `>>`                 | 当前行向右缩进一次         | 用于手动调整单行。                                           |
| `<<`                 | 当前行向左缩进一次         | 用于手动调整单行。                                           |
| `==`                 | 自动对齐当前行             | 按照语法逻辑自动缩进。                                       |
| `v` + 选中代码 + `=` | **自动格式化选中的代码块** | 按 `v` 进入可视模式，选中多行乱码，按 `=` 瞬间整齐。         |
| `gg=G`               | **自动格式化整个文件**     | **终极排版键**：从头(`gg`)到尾(`G`)执行格式化(`=`)，一键拯救全篇代码格式。 |

## 4. 高效编辑的“杀手锏”（文本对象）

这是 Vim 最强大的功能之一，它能理解代码的结构（比如引号和括号内部的内容），并进行整块修改。

| 命令  | 操作结果                                   | 实战场景                                                     |
| ----- | ------------------------------------------ | ------------------------------------------------------------ |
| `ci"` | 清除双引号 `""` 内的内容，并进入插入模式   | 需要修改 `printf("hello");` 里面的字符串时，光标在引号内按 `ci"`。 |
| `ci{` | 清除大括号 `{}` 内的内容，并进入插入模式   | 需要重写整个 `while` 或 `函数` 内部的逻辑时。                |
| `di{` | 删除大括号 `{}` 内的内容，但保持普通模式   | 纯粹删除代码块内部内容。                                     |
| `cw`  | 删除光标当前位置到单词末尾，并进入插入模式 | 快速修改一个变量名。                                         |

## 5. 块操作（批量注释/修改）

当需要同时操作多行（比如批量加注释）时，这是必备技能。

| 命令组合   | 操作步骤                          | 实战场景：批量添加注释 `//`              |
| ---------- | --------------------------------- | ---------------------------------------- |
| `Ctrl + v` | 进入**可视块模式** (Visual Block) | 1. 光标放在第一行开头，按 `Ctrl+v`。<br> |

<br>2. 按方向键或 `j` 向下选中多行的行首。<br>

<br>3. 按大写 `I` 进入插入模式。<br>

<br>4. 输入 `//`。<br>

<br>5. **按两次 `Esc**`，所有选中行都会加上注释。 |

> **考试小贴士**：在紧张的考试环境中，如果因为误触不知道自己处于什么模式，疯狂按几下 `Esc` 键永远是最安全的回到普通模式的办法。然后敲击 `:w` 随时保存你的进度。
