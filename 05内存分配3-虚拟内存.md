
### 1. 基础概念引入
*   **地址空间 vs 存储空间**：地址空间大小和存储空间大小没有必然联系。地址字长（指针长度）取决于硬件容量（如32bit、64bit）和软件需求。
*   **程序、进程和作业**（初步）
    *   **程序**: 是静止的磁盘可执行文件；程序是**静态且长久**的
    *   **进程**：进程是**竞争资源**的基本单位，能真实描述并发，有诞生消亡的短暂**生命周期**，且可**创建其他进程**；是动态的、分配资源的基本单位（分为系统进程和用户进程）；进程由PCB、程序和数据集合组成。
        *   进程是对已提交程序执行的描述，只要创建就有一部分在内存中。
        *   一个进程可以包含多个程序，同一个程序也可存在于多个进程中
    *   **作业**：作业是要求计算机的**工作的集合**，作业包括程序、数据和操作说明书；一个作业可划分为若干进程来完成。

### 2. 分页式存储管理基本原理
*   **基本思想**：把**逻辑地址连续的程序**分散存放到若干**不连续**的内存区域内。目的是为了解决动态内存分配、碎片和紧凑问题。
*   **核心概念**：
    *  VA: **页（页面）**：逻辑地址空间被分成**大小相等**的块。
    *  PA: **存储块（页框/Frame）**：主存空间被分为与页面同等大小的片。
*   **纯分页系统**：如果不具备页面置换功能，程序的所有页必须一次性装入主存。
    *   **优点**：没有外部碎片（内碎片不超过页大小），程序不必连续存放，便于动态扩展。
    *   **缺点**：程序必须全部装入内存。

### 3. 地址结构与内存分配
*   **地址结构**：逻辑地址结构 = 页号 Page num + 位移量 offset。
    *   如果逻辑地址空间为 $2^m$，页大小为 $2^n$（通常为2的幂，现代OS常用4KB），则高 $m-n$ 位为页号，低 $n$ 位为页偏移。
    *   **计算公式**：给定逻辑地址 A 和页面大小 L，页号 $P = INT(A / L)$（保留整数），页内地址 $d = A \pmod L$。
*   物理地址结构 = 块号 + 块内位移 
*   **内存分配基本思想**：以页为单位，按进程大小分配。逻辑上相邻的页，物理上不一定相邻。

### 4. 数据结构——页表与地址变换 【计组】
*   **数据结构**：包含**进程页表(page table)**（记录**逻辑页号到物理页号的映射（数组）**）、物理页面表（位示图/链表记录内存分配）以及请求表。
*   **页表**：存放在内存中，用于记录分配情况和实现动态重定位。通过页表寄存器给出基址和长度。访问数据需**2次访存**（查页表1次，访存1次）。
*   **地址变换机构**：
    1.  **越界保护**：将页号与页表长度比较，产生越界中断。
    2.  **地址变换**：页表定位（页表始址 + 页号 × 页表项长度）读出物理块号。
    3.  **合并**：拼接块号和页内地址得到物理地址。
*   **页表项 (PTE) 内容**：包含修改位 (Modified/dirty bit)、访问位 (Referenced bit)、在/不在位 (Present/Absent bit) 以及保护位 (Protection bits)。

### 5. 页面大小的选取与多级页表
*   **页面大小选取**：
    *   **较小页面**：优点是内存碎片少，利用率高；缺点是页表长占用大，页面换入换出速度低（磁盘IO多）。
    *   **较大页面**：优点是页表短，换入换出快；缺点是页内碎片大。
    *   **最优公式**：最优页面大小 $p = \sqrt{2es}$（$e$ 为页表项大小，$s$ 为进程平均占用字节）。
*   **多级页表**：解决一级页表在逻辑地址空间大时（如32位系统4KB页面需4MB连续页表空间）占用巨大连续内存的问题。
    *   将页表进行分页（外部页表/页目录），离散存放，按需动态调入内部页表。

### 6. 快表 (TLB) 与访存性能优化
*   **性能问题**：分页机制导致访存次数增加（如二级页表机制需3次访存），效率严重下降。
*   **MMU与快表 (TLB)**：
    *   引入存储管理单元 (MMU)，内部包含 TLB（高速缓存，专门针对页表项）、TLB控制单元和页表查找单元。
    *   **工作过程**：CPU生成虚拟地址 -> 查TLB -> 若命中(TLB hit)直接出物理地址 -> 若未命中(TLB miss)查外部页表并置换进TLB -> 查权限 -> 映射出物理地址（未映射则page fault）。
    *   **ASID特性**：部分TLB支持地址空间标识码 (ASID) 来唯一标识进程。若不支持，上下文切换时必须清空(flush) TLB。
    *   **访存层级**：CPU -> TLB Lookup -> Cache -> Main Memory。

### 7. 反置页表 (Inverted Page Table)
*   **背景**：常规页表按逻辑页号组织，耗费大量内存。
*   **概念**：反置页表依据进程在**物理页面号**来组织，大小只与物理内存大小相关（与逻辑空间大小无关）。
*   **表项内容**：包含逻辑页号 P 和所属进程标志符 pid。
*   **地址变换与改进**：需用 pid 和 p 检索页表，匹配后的表项序号即为物理块号。因为按物理地址排序，检索慢，通常结合**哈希页表 (Hash Table)** 来加速查找。
*   **缺点**：采用反置页表的系统很难实现内存共享，因为每个物理帧只对应一个虚拟页条目。

### 8. 页共享与保护
*   **保护方式**：通过地址越界保护和页表中的保护位（只读、读写、执行）实现。
*   **页共享**：不同进程的页表指向相同的物理块即可实现共享。但若共享数据与非共享数据在同一页内，会导致非共享数据也被暴露。实现数据共享的最好方法是**分段存储管理**。
