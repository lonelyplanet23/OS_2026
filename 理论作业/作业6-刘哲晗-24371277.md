### 问题2

I/O 控制方式主要有以下四种。按 **CPU 干预程度从高到低** 排序为：

1. **程序直接控制方式 / 程序查询方式**：CPU 主动查询设备状态并搬运数据，CPU 干预最多。
2. **中断驱动方式**：CPU 发出 I/O 请求后可以去执行其他任务，设备完成或需要服务时再中断 CPU。
3. **DMA 方式**：数据块在设备和内存之间由 DMA 控制器直接传送，CPU 只负责初始化和结束处理。
4. **通道控制方式**：由专门的通道处理机执行 I/O 程序，可管理更复杂的一组 I/O 操作，CPU 干预最少。

### 问题3

#### 场景A：读取键盘输入的单个字符

选择：**中断驱动方式**。

理由：键盘输入速度慢，且字符到达时间不确定。如果 CPU 一直轮询键盘状态，会浪费大量 CPU 时间。采用中断方式时，只有用户按键、字符到达时才通知 CPU，效率更高。

#### 场景B：从磁盘读取一个 4KB 的文件块到内存

选择：**DMA 方式**。

理由：磁盘块读写属于成块数据传输。如果每个字节都由 CPU 搬运，开销较大。DMA 可以让数据在磁盘控制器和内存之间直接传送，CPU 只需设置源地址、目的地址、长度等参数，完成后再处理中断。

#### 场景C：连续打印 100 页文档，数据已提前准备好

选择：**通道控制方式**。

理由：连续打印 100 页属于长时间、连续的外设输出任务。如果由 CPU 频繁干预，会降低系统并发性。通道控制方式可以让专门的 I/O 通道按照预先给定的 I/O 命令序列持续控制打印过程，CPU 只需启动任务并在关键事件发生时处理结果，适合大量连续 I/O。

### 问题4

#### (1) 更换磁盘接口后的影响

如果系统管理员将用户的磁盘从 `sda` 接口更换为 `nvme0n1`：

- 使用物理命名 `/dev/sda1/myfile.txt` 的程序可能会失效。因为路径中直接包含了设备名 `sda1`，设备更换后原路径可能不存在，或者指向了错误的设备。
- 使用逻辑命名 `/home/user/myfile.txt` 的程序通常不受影响。只要系统管理员把新的设备正确挂载到 `/home` 或相应目录，用户程序仍然可以通过同一个逻辑路径访问文件。

#### (2) 推荐的命名方式

从设备独立性的角度看，操作系统应该推荐用户使用 **逻辑命名方式**。

原因：逻辑命名把用户看到的文件路径与底层具体设备分离。用户程序不需要关心文件实际位于哪块磁盘、哪个接口、哪个控制器上。这样即使底层硬件更换、磁盘迁移或重新挂载，只要逻辑目录结构保持不变，应用程序就不需要修改，提高了系统的可维护性和可移植性。

### 问题5

已知：

- 磁盘读入一块到缓冲区时间：\(T = 4\text{ ms}\)
- 缓冲区数据传送到用户区时间：\(M = 1\text{ ms}\)
- CPU 处理一块数据时间：\(C = 2\text{ ms}\)
- 共处理 \(n = 10\) 块数据


#### 1. 单缓冲

单缓冲时，基本过程为：

1. 第 1 块先从磁盘读入缓冲区，需要 \(T\)。
2. 然后把该块从缓冲区送入用户区，需要 \(M\)。
3. CPU 处理该块需要 \(C\)，同时磁盘可以读下一块到缓冲区。
4. 但由于只有一个缓冲区，下一次“缓冲区到用户区”的传送要等到下一块读入完成，并且用户区处理关系满足顺序要求。

因此，中间每增加一块的流水周期为：

\[
\max(T, C) + M
\]

本题中：

\[
\max(T, C) = \max(4, 2) = 4\text{ ms}
\]

所以 10 块的总时间为：

\[
T + M + (n - 1)[\max(T, C) + M] + C
\]

代入数据：

\[
4 + 1 + 9 \times (4 + 1) + 2 = 4 + 1 + 45 + 2 = 52\text{ ms}
\]

单缓冲处理完 10 块数据的总时间为 52 ms。



#### 2. 双缓冲

双缓冲时，磁盘可以在一个缓冲区中读入下一块数据，同时 CPU 对另一个缓冲区对应的数据进行“传送到用户区 + 处理”。

因此，中间的流水周期为：

\[
\max(T, M + C)
\]

本题中：

\[
M + C = 1 + 2 = 3\text{ ms}
\]

\[
\max(T, M + C) = \max(4, 3) = 4\text{ ms}
\]

总时间为：

\[
T + (n - 1)\max(T, M + C) + M + C
\]

代入数据：

\[
4 + 9 \times 4 + 1 + 2 = 43\text{ ms}
\]

所以，**双缓冲处理完 10 块数据的总时间可减少为 43 ms**。



结论：

- 单缓冲：**52 ms**
- 双缓冲：**43 ms**
- 减少时间：\(52 - 43 = 9\text{ ms}\)

### 问题6

SPOOLing 技术能把独占设备变成共享设备，是因为它在慢速独占设备和多个用户进程之间增加了一个 **高速磁盘上的输入/输出井**，把用户进程对设备的直接占用转化为对磁盘文件队列的提交。

以多个用户进程同时请求打印为例：

1. 打印机本身是独占设备，同一时刻只能打印一个进程的输出。
2. 如果没有 SPOOLing，某个进程获得打印机后，其他进程只能等待打印机释放。
3. 使用 SPOOLing 后，每个用户进程并不直接占用打印机，而是把要打印的数据先写入磁盘上的打印输出井，形成一个打印作业。
4. 系统中的 SPOOLing 进程或打印守护进程按照队列顺序，把这些打印作业依次送给真正的打印机。
5. 对用户进程来说，它只要把打印内容提交到输出井就可以继续执行，不需要长期等待真实打印机完成打印。

因此，SPOOLing 的本质是利用**磁盘缓冲**和作**业队列**，把多个进程对独占设备的竞争转化为对共享磁盘文件的排队访问，使用户感觉打印机可以被多个进程同时共享。

### 问题7

同步 I/O 与异步 I/O 的核心区别在于：**发出 I/O 请求后，调用进程是否需要等待 I/O 操作完成，以及 I/O 完成后由谁通知或继续处理结果**。

按题目给出的两个阶段分析：

- 阶段1：数据从设备准备到内核缓冲区。
- 阶段2：数据从内核缓冲区拷贝到用户空间。

#### 同步 I/O

同步 I/O 中，用户进程发起 I/O 调用后，通常要等待 I/O 操作完成后才能继续执行。

具体来说：

1. 在阶段1，进程可能被阻塞，等待设备把数据准备好并送到内核缓冲区。
2. 在阶段2，内核还需要把数据从内核缓冲区拷贝到用户空间，调用进程通常也要等待这一过程完成。
3. 当阶段1和阶段2都完成后，系统调用返回，用户进程才能直接使用数据。

因此，同步 I/O 的特点是：**调用返回时，数据通常已经在用户空间中可用**。

#### 异步 I/O

异步 I/O 中，用户进程发起 I/O 请求后不需要一直等待，可以继续执行其他计算。内核在后台完成 I/O 操作，并在完成后通过信号、回调、事件通知等方式告知用户进程。

具体来说：

1. 阶段1由内核和设备在后台完成，用户进程不必阻塞等待设备准备数据。
2. 阶段2也由内核负责安排，把数据从内核缓冲区拷贝到用户空间。
3. 当两个阶段都完成后，内核再通知用户进程：本次 I/O 已完成，数据可以使用。

因此，异步 I/O 的特点是：调用很快返回，真正的 I/O 完成通过后续通知体现。

### 拓展思考题

现代 NVMe 固态硬盘采用多个中断向量 MSI-X，相比传统单中断向量，在多核系统中有明显性能优势。

主要体现在：

1. **减少中断竞争**  
   单中断向量下，所有 I/O 完成事件都可能集中发送到同一个 CPU 核心，导致该核心中断负载过重。多个 MSI-X 向量可以把不同队列的中断分散到不同 CPU 核心。

2. **提高并行处理能力**  
   NVMe 本身支持多提交队列和多完成队列。每个 CPU 核心可以拥有或主要处理自己的 I/O 队列，并通过对应的中断向量接收完成通知，从而减少跨核同步，提高并行度。

3. **改善缓存局部性**  
   如果某个核心提交 I/O 请求，并且完成中断也尽量回到同一个核心处理，那么相关数据结构更可能保留在该核心缓存中，减少缓存失效和跨核数据迁移。

4. **降低中断处理延迟**  
   单中断向量容易造成中断排队。当 I/O 请求数量很大时，多个中断向量可以让多个核心同时处理完成事件，降低平均响应时间。

5. **提升高 IOPS 场景下的可扩展性**  
   NVMe SSD 的 IOPS 很高，如果仍然使用传统单中断向量，CPU 中断处理可能成为瓶颈。MSI-X 与多队列机制结合后，可以更好地匹配多核 CPU 和高速 SSD 的并行能力。

因此，多个 MSI-X 中断向量的核心优势是：把原本集中在单个 CPU 核心上的 I/O 完成通知分散到多个核心处理，减少瓶颈，提高多核系统中的并行性、缓存局部性和整体 I/O 吞吐量。
