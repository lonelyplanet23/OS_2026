## 生产者－消费者问题

**问题描述**：若干进程通过有限的共享缓冲区交换数据。其中，"生产者"进程不断写入，而"消费者"进程不断读出；共享缓冲区共有N个；任何时刻只能有一个进程可对共享缓冲区进行操作。

**两个隐含条件**：
1. 消费者和生产者数量不固定。
2. 消费者和生产者不能同时使用缓存区。

**行为分析**：
- 生产者：生产产品，放置产品(有空缓冲区)。
- 消费者：取出产品(有产品)，消费产品。

**行为关系**：
- 生产者之间：互斥(放置产品)
- 消费者之间：互斥(取出产品)
- 生产者与消费者之间：互斥(放/取产品) + 同步(放置——取出)

---

## 用PV操作解决生产者/消费者问题

**信号量设置**：
```c
semaphore mutex = 1;  // 互斥访问缓冲区
semaphore empty = N;  // 空闲缓冲区数量
semaphore full = 0;   // 产品数量
```

> full + empty == N

---

## PV操作代码

**生产者**：
```c
P(empty);   // 申请空缓冲区
P(mutex);   // 互斥访问
one >> buffer  // 放入产品
V(mutex);
V(full);    // 增加产品计数
```

**消费者**：
```c
P(full);    // 申请产品
P(mutex);   // 互斥访问
one << buffer  // 取出产品
V(mutex);
V(empty);   // 增加空缓冲区
```

> 思考：如果两个V交换顺序呢？

## 完整的伪代码

```c
Semaphore full = 0;
Semaphore empty = n;
Semaphore mutex = 1;
ItemType buffer[0..n-1];
int in = 0, out = 0;
```

**Producer**：
```c
producer() {
    while(true) {
        生产产品nextp;
        P(empty);
        P(mutex);
        buffer[in] = nextp;
        in = (in + 1) MOD n;
        V(mutex);
        V(full);
    }
}
```

**Consumer**：
```c
consumer() {
    while(true) {
        P(full);
        P(mutex);
        nextc = buffer[out];
        out = (out + 1) MOD n;
        V(mutex);
        V(empty);
        消费nextc中的产品
    }
}
```

## 用管程解决生产者消费者问题

```c
monitor ProducerConsumer
    condition full, empty;
    integer count;

    procedure insert(item: integer);
    begin
        if count == N then wait(full);
        insert_item(item);
        count++;
        if count == 1 then signal(empty);
    end;

    function remove: integer;
    begin
        if count == 0 then wait(empty);
        remove = remove_item;
        count--;
        if count == N-1 then signal(full);
    end;

    count := 0;
end monitor;
```

**使用方式**：
```c
procedure producer;
begin
    while true do
    begin
        item = produce_item;
        ProducerConsumer.insert(item);
    end
end

procedure consumer;
begin
    while true do
    begin
        item = ProducerConsumer.remove;
        consume_item(item);
    end
end
```

> 采用AND信号量集：SP(empty, mutex), SP(full, mutex) —— 同学们可自行练习

## 读者－写者问题

**问题描述**：对共享资源的读写操作，任一时刻"写者"最多只允许一个，而"读者"则允许多个，即：
- **"读－写"互斥
- **"写－写"互斥
- **"读－读"允许**

**生活中的实例**：12306订票
- 读者：查询余票
- 写者：订票/退票

**规则**：
- 同一时刻，只有一个激活的写进程
- 同一时刻，可以有多个激活的读进程

---

## 采用信号量机制

**基本框架**：

```
Writer                      Reader
P(wmutex);                  if first_reader? then P(wmutex);
写数据                       读数据
V(wmutex);                  if last_reader? then V(wmutex);
```

**信号量与变量**：
- `wmutex`：表示"允许写"，初值是1。
- `readcount`：表示"正在读"的进程数，初值是0；
- `mutex`：表示对readcount的互斥操作，初值是1。

---

## 读者优先算法

| Writer | Reader |
|--------|--------|
| P(wmutex); | P(mutex); |
| write | if readcount=0 then P(wmutex); |
| V(wmutex); | readcount := readcount + 1; |
| | V(mutex); |
| | read |
| | P(mutex); |
| | readcount := readcount - 1; |
| | if readcount=0 then V(wmutex); |
| | V(mutex); |

**思考题**：
- 如果有一个写者，第一个读者将阻塞，其他读者也会阻塞
- 如果一个读者激活，所有的读者都会陆续激活，哪个最先激活？
- 最后一个读者退出时，唤醒一个写者；如果没有写者呢？
- 如果读者和写者均阻塞在wmutex上，写者退出后，哪一个先激活？

> 该算法是对读者有利，还是对写者有利？→ **读者有利**

---

## 对写者有利的算法

**写者规则**：
- 开始后，没有其他写者、读者可以进入
- 必须确保没有正在读的读者，才能开始写
- 写完之后允许其他读写者进入

**读者规则**：
- 第一个读者开始读，到最后一个读者结束读之间不允许有写进程进入写过程

**写者优先的改进**：
- 写者出现后，新读者不允许抢在该写者之前进行读操作。
- 也就是说：写者出现后，后续读者不允许执行rcount=rcount+1
- 因此需要增加一个互斥信号量

---

## 读写公平的算法

增加信号量`rwmutex`控制读写公平：

| Reader | Writer |
|--------|--------|
| P(rwmutex); | P(rwmutex); |
| P(mutex); | P(wmutex); |
| if readcount=0 then P(wmutex); | 写数据 |
| readcount := readcount + 1; | V(wmutex); |
| V(mutex); | V(rwmutex); |
| V(rwmutex); // 注意这个位置 | |
| read | |
| P(mutex); | |
| readcount := readcount - 1; | |
| if readcount=0 then V(wmutex); | |
| V(mutex); | |

> 思考：V(rwmutex) 能不能放在read之后？

## 哲学家进餐问题

**问题描述**：（由Dijkstra首先提出，Tony Hoare重新表述为哲学家就餐问题）

5个哲学家围绕一张圆桌而坐，桌子上放着5支筷子，每两个哲学家之间放一支；

哲学家的动作包括思考和进餐：
- 进餐时需要同时拿起他左边和右边的两支筷子
- 思考时则同时将两支筷子放回原处。

**问题**：如何保证哲学家们的动作正确有序进行？
- 不出现相邻者同时进餐；
- 不出现有人永远拿不到足够筷子，从而无法进餐（死锁/饿死）。

---

## 基于P-V操作的方法（有问题）

```c
Var chopstick : array[0..4] of semaphore;  // 初值均为1

Philosopher i:
    P(chopstick[i]);           // 拿左边筷子
    P(chopstick[(i+1) mod 5]); // 拿右边筷子
    eat                         // 进餐
    V(chopstick[i]);
    V(chopstick[(i+1) mod 5]);
    think                       // 思考
```

**问题**：5个哲学家同时拿起左边筷子 → 死锁！

---

## 哲学家就餐问题的解题思路

**方案1**：至多只允许四个哲学家同时（尝试）拿起筷子进餐，以保证至少有一个哲学家能够进餐，最终总会释放出他所使用过的两支筷子，从而可使更多的哲学家进餐。
- 设置信号量room=4。

**方案2**：对筷子进行编号，奇数号先拿左，再拿右；偶数号相反。

**方案3**：同时拿起两根筷子，否则不拿起（AND信号量集）。

---

## 方案3：基于AND信号量集的方法

```c
Var chopstick : array[0..4] of semaphore;

Philosopher i:
    think                           // 思考
    SP(chopstick[(i+1) mod 5], chopstick[i]);  // 同时申请两根筷子
    eat                             // 进餐
    SV(chopstick[(i+1) mod 5], chopstick[i]);  // 同时释放两根筷子
```

## "生产者-消费者"扩展问题

**问题**：设有一个可以装A、B两种物品的仓库,其容量无限大,但要求仓库中A、B两种物品的数量满足下述不等式:

> -M ≤ A物品数量 - B物品数量 ≤ N

即：
- A不能比B多N以上
- B不能比A多M以上

其中M和N为正整数。

试用信号量和PV操作描述A、B两种物品的入库过程。

---

## 算法描述

```c
sa: A货物比B货物多的额度
sb: B货物比A货物多的额度

Semaphore mutex = 1, sa = N, sb = M;

cobegin

procedure A:
    while(TRUE)
    begin
        p(sa);        // A入库会减少A-B的额度
        p(mutex);
        A产品入库；
        V(mutex);
        V(sb);        // A入库增加了B-A的额度
    end

procedure B:
    while(TRUE)
    begin
        p(sb);
        p(mutex);
        B产品入库;
        V(mutex);
        V(sa);
    end

coend
```

## 理发师问题

**问题描述**：理发店里有1位理发师、1把理发椅和n把供等候理发的顾客坐的椅子；

- 如果没有顾客，理发师便在理发椅上睡觉，当一个顾客到来时，叫醒理发师；
- 如果理发师正在理发时，又有顾客来到，则如果有空椅子可坐，就坐下来等待，否则就离开。

**互斥资源**：理发师、顾客、椅子

**同步约束**：访问椅子、顾客唤醒理发师、理发师唤醒下一位等待顾客

```c
int waiting = 0;  // 等待顾客数

Barber:
while(true) {
    睡觉
    开始理发
    理发
}

Custom_i:
if (waiting < N) {
    坐下来
    唤醒理发师
    等待理发
    接受理发
}
```

---

## 理发师问题变种

1. **多理发师**：理发店里有m位理发师，n把供等候理发的顾客坐的椅子；如果没有顾客，理发师便睡觉，当顾客到来时，叫醒1位理发师；如果所有理发师正在理发时，又有顾客来到，则如果有空椅子可坐，就坐下来等待，否则就离开。

2. **店满等待**：理发店里有m位理发师，n把供等候理发的顾客坐的椅子；如果没有顾客，理发师便睡觉，当顾客到来时，叫醒1位理发师；如果所有理发师正在理发时，又有顾客来到，则如果有空椅子可坐，就坐下来等待，否则就在门口等待。
