# 课程讲稿整理经验总结

## 任务描述
根据课堂录音转写的原始txt文稿（带时间戳），整理成结构清晰、适合阅读和上传分析的markdown讲稿。

---

## 整理原则

### 1. 保留原汁原味
- 保留老师讲课的所有核心知识点
- 保留例题、思考题、强调的考点
- 不要删减重要内容，只调整结构

### 2. 结构清晰化
- 按照讲课顺序分章节
- 每节有明确标题
- 使用三级标题体系：
  ```
  # 课程大标题
  ## 第一节：XXX
  ### 1. XXX
  ```

### 3. 表格化对比
- 对比性内容（不同算法、不同方案）一定要用表格
- 方便读者一目了然
- 常见对比：
  - 硬件做什么 vs 软件做什么
  - 算法A vs 算法B 的优缺点
  - 不同地址空间段的映射方式

### 4. 突出考点
- 老师明确说"这个考试常考"、"请大家记住"的内容，要突出
- 计算题做题方法总结成步骤

### 5. 解决口语化
- 去掉重复的口头语
- 把断续的句子整理通顺
- 保留讲课的语气，不要太生硬

---

## 工作流程

1. **分段读取**：大文件分段读取，避免token超出限制
2. **理解内容**：识别出课程主题、知识点脉络
3. **重新组织**：按逻辑重新分章节
4. **添加格式**：
   - 标题层级
   - 代码块（C语言例子）
   - 表格对比
   - 公式（使用markdown LaTeX）
5. **总结**：课程结尾做总结，点出重点

---

## 针对操作系统课程的特定经验

### 常见考点整理优先级
1. **算法题**：页面置换算法计算缺页次数 → 一定要总结做题步骤
2. **对比题**：分页vs分段、FIFO vs LRU → 一定要表格对比
3. **概念题**：Belady异常、缺页处理流程 → 步骤清晰列出
4. **实现题**：自映射为什么解决鸡生蛋问题 → 讲清楚逻辑链

### 公式格式
- 页大小计算：`\(2^{12} = 4096 = 4KB\)`
- 地址拆分：`10 + 10 + 12 = 32` （二级页表）

### 实验相关内容
- 老师插播的C语言、MIPS布局内容也要保留，对实验帮助很大
- 单位换算、地址对齐这些基本功要单独总结
