# 外部中断/异常处理全流程详解

## 整体流程概览

```
【外部中断/异常发生】
        |
        v
【异常处理入口 entry.S】 ── SAVE_ALL 保存现场
        |                                   ↳ 内核态/用户态判断，栈指针切换
        v
【异常分发 genex.S】 ── 根据CAUSE寄存器跳转到对应handler
        |
        +------------------------+------------------------+
        |                        |                        |
        v                        v                        v
【TLB异常】              【时钟中断/INT】          【系统调用/Syscall】
handle_tlb              handle_int               handle_sys
    |                      |                           |
    v                      v                           v
do_tlb_refill()       schedule()调度            do_syscall()处理
    |                      |                           |
    |                      v                           |
    |              【切换进程 env_run】                 |
    |                      |                           |
    +------------------------+------------------------+
        |
        v
【RESTORE_ALL 恢复现场 ret_from_exception】
        |
        v
   【eret 返回】
```

## 第一阶段：异常入口与现场保存 (entry.S)

### 1.1 异常发生时的硬件行为

当异常/中断发生时，MIPS CPU自动完成：
1. 设置 **CP0_CAUSE** 寄存器的异常码（ExcCode）标识异常类型
2. 将异常发生时的 **PC** 保存到 **CP0_EPC**
3. 设置 **CP0_STATUS** 的 **EXL** 位（Exception Level），进入内核态
4. PC跳转到异常入口地址（`0x80000000`）

### 1.2 SAVE_ALL 宏详解 ([stackframe.h:6-66](lab-solution/mips32/include/stackframe.h#L6-L66))

这是异常处理的**第一步**，也是最关键的一步：

```asm
.macro SAVE_ALL
.set noat
.set noreorder
    mfc0    k0, CP0_STATUS
    andi    k0, STATUS_UM       // 检查是否来自用户态
    beqz    k0, 1f              // 如果是内核态，跳转
    move    k0, sp              // 保存用户栈指针
    
    li      sp, KSTACKTOP       // 用户态进入：切换到内核栈 KSTACKTOP
1:                              // 内核态进入：sp 已经是内核栈
    subu    sp, sp, TF_SIZE     // 在栈上分配 Trapframe 空间
    sw      k0, TF_REG29(sp)    // 保存原来的sp
    
    // 保存 CP0 特殊寄存器
    mfc0    k0, CP0_STATUS
    sw      k0, TF_STATUS(sp)
    mfc0    k0, CP0_CAUSE
    sw      k0, TF_CAUSE(sp)
    mfc0    k0, CP0_EPC
    sw      k0, TF_EPC(sp)
    // ... 保存 HI, LO, BADVADDR ...
    
    // 保存所有通用寄存器 $0-$31
    sw      $0, TF_REG0(sp)
    sw      $1, TF_REG1(sp)
    // ... 共32个寄存器 ...
.endm
```

**关键点**：
- **栈位置**：Trapframe 固定放在 **KSTACKTOP** 下方！这是整个系统的约定
- **内核态 vs 用户态**：
  - **用户态进入**：STATUS_UM=1 → 将 sp 从用户栈切换到内核栈 KSTACKTOP
  - **内核态进入**：STATUS_UM=0 → 直接使用当前内核栈，不切换
- **栈生长方向**：从高地址向低地址生长，`sp = KSTACKTOP - TF_SIZE`

### 1.3 进入内核模式准备 ([entry.S:17-19](lab-solution/mips32/kern/entry.S#L17-L19))

```asm
mfc0    t0, CP0_STATUS
and     t0, t0, ~(STATUS_UM | STATUS_EXL | STATUS_IE)
mtc0    t0, CP0_STATUS
```

- 清除 UM 位：确保处于内核态
- 清除 EXL 位：允许嵌套异常（EPC可被新异常更新）
- 清除 IE 位：全局关中断，保证异常处理原子性

## 第二阶段：异常分发 (entry.S + genex.S)

### 2.1 根据异常码跳转 ([entry.S:21-24](lab-solution/mips32/kern/entry.S#L21-L24))

```asm
mfc0    t0, CP0_CAUSE
andi    t0, 0x7c              // 提取 ExcCode 字段 (bits 6-2)
lw      t0, exception_handlers(t0)  // 查表获取handler地址
jr      t0                    // 跳转到对应handler
```

**异常码表**（部分）：
| 异常码 | 类型 | Handler |
|--------|------|---------|
| 0 | Int（外部中断） | handle_int |
| 1 | TLB Mod | handle_mod |
| 2 | TLB Load | handle_tlb |
| 3 | TLB Store | handle_tlb |
| 8 | Syscall | handle_sys |

### 2.2 BUILD_HANDLER 宏 ([genex.S:4-12](lab-solution/mips32/kern/genex.S#L4-L12))

统一的异常处理函数生成模板：

```asm
.macro BUILD_HANDLER exception handler
NESTED(handle_\exception, TF_SIZE + 8, zero)
    move    a0, sp           // a0 = Trapframe 指针（参数）
    addiu   sp, sp, -8       // 为函数调用分配栈空间
    jal     \handler         // 调用C函数
    addiu   sp, sp, 8
    j       ret_from_exception  // 统一返回点
END(handle_\exception)
.endm
```

**示例**：`BUILD_HANDLER tlb do_tlb_refill` 生成 `handle_tlb` 函数

## 第三阶段：分情况处理

### 情况 A：TLB 异常 - 不切换进程

#### 3.A.1 do_tlb_refill 流程 ([tlbex.c:48-67](lab-solution/mips32/kern/tlbex.c#L48-L67))

```c
void _do_tlb_refill(u_long *pentrylo, u_int va, u_int asid) {
    tlb_invalidate(asid, va);
    
    // 循环查找页表，找不到则被动分配
    while (page_lookup(cur_pgdir, va, &ppte) == NULL) {
        passive_alloc(va, cur_pgdir, asid);  // 分配物理页
    }
    
    // 填充 TLB EntryLo0/EntryLo1
    pentrylo[0] = ppte[0] >> 6;
    pentrylo[1] = ppte[1] >> 6;
}
```

**关键特征：不切换进程**
- 只补全 TLB 页表项
- 当前进程的 `curenv` 不变
- **$sp 指针不动**，仍指向原来的 Trapframe

#### 3.A.2 返回路径

```
do_tlb_refill 返回
        |
        v
j ret_from_exception
        |
        v
RESTORE_ALL 恢复 Trapframe 到寄存器
        |
        v
eret 指令 ← CP0_EPC 指向**异常发生的指令**
        |
        v
回到原进程，重新执行那条指令
```

### 情况 B：时钟中断 - 触发进程切换

#### 3.B.1 handle_int ([genex.S:20-29](lab-solution/mips32/kern/genex.S#L20-L29))

```asm
NESTED(handle_int, TF_SIZE, zero)
    mfc0    t0, CP0_CAUSE
    mfc0    t2, CP0_STATUS
    and     t0, t2
    andi    t1, t0, STATUS_IM7   // 检查时钟中断
    bnez    t1, timer_irq
timer_irq:
    li      a0, 0                 // yield=0
    j       schedule              // 进入调度器
END(handle_int)
```

#### 3.B.2 schedule() 调度器 ([sched.c:17-51](lab-solution/mips32/kern/sched.c#L17-L51))

```c
void schedule(int yield) {
    static int count = 0;  // 当前进程剩余时间片
    
    // 需要切换：时间片用完 或 yield 或 当前进程不可运行
    if (yield || count == 0 || e == NULL || e->env_status != ENV_RUNNABLE) {
        // 把当前进程移到队列尾部（如果还能运行）
        if (e != NULL && e->env_status == ENV_RUNNABLE) {
            TAILQ_REMOVE(&env_sched_list, e, env_sched_link);
            TAILQ_INSERT_TAIL(&env_sched_list, e, env_sched_link);
        }
        // 取队首新进程
        e = TAILQ_FIRST(&env_sched_list);
        count = e->env_pri;  // 重置时间片
    }
    count--;
    env_run(e);  // 切换到新进程
}
```

#### 3.B.3 env_run() 进程切换 ([env.c:474-510](lab-solution/mips32/kern/env.c#L474-L510))

这是**真正发生进程切换**的地方！

```c
void env_run(struct Env *e) {
    // Step 1: 保存旧进程上下文（如果不是第一次运行）
    if (curenv) {
        // 关键！Trapframe 就在 KSTACKTOP-1 位置
        curenv->env_tf = *((struct Trapframe *)KSTACKTOP - 1);
    }
    
    // Step 2: 切换当前进程指针
    curenv = e;
    
    // Step 3: 切换页表（地址空间切换）
    cur_pgdir = curenv->env_pgdir;
    
    // Step 4: 加载新进程的 Trapframe
    env_pop_tf(&curenv->env_tf, curenv->env_asid);
}
```

#### 3.B.4 env_pop_tf ([env_asm.S:7-14](lab-solution/mips32/kern/env_asm.S#L7-L14))

```asm
LEAF(env_pop_tf)
    mtc0    a1, CP0_ENTRYHI   // a1 = ASID，切换地址空间
    move    sp, a0            // 关键！sp 现在指向新进程的 env_tf
    RESET_KCLOCK
    j       ret_from_exception  // 进入统一返回路径
END(env_pop_tf)
```

**这里的魔法**：
- `sp` 被**重新赋值**为新进程的 `env_tf` 地址！
- 接下来的 `RESTORE_ALL` 会从**新进程的 Trapframe** 恢复寄存器

## 第四阶段：RESTORE_ALL 与返回 ([genex.S:16-18](lab-solution/mips32/kern/genex.S#L16-L18))

### 4.1 RESTORE_ALL 宏 ([stackframe.h:68-110](lab-solution/mips32/include/stackframe.h#L68-L110))

```asm
.macro RESTORE_ALL
    lw      v0, TF_STATUS(sp)
    mtc0    v0, CP0_STATUS      // 恢复 STATUS（含 UM/IE 位）
    
    // 恢复 HI, LO, EPC ...
    
    // 恢复所有通用寄存器
    lw      $31, TF_REG31(sp)
    lw      $30, TF_REG30(sp)
    // ... $1-$28 ...
    
    lw      sp, TF_REG29(sp)    // 最后恢复sp，栈自动"释放"
.endm
```

**注意顺序**：
- sp 是最后恢复的！这保证在恢复期间能正确访问栈上的 Trapframe

### 4.2 eret 指令

```asm
ret_from_exception:
    RESTORE_ALL
    eret  // 异常返回：CP0_EPC → PC，自动清除 EXL 位
```

## 核心对比总结

| 维度 | TLB 异常（不切换） | 时钟中断（切换进程） |
|------|-------------------|-------------------|
| **sp 变化** | sp 不变，始终指向 KSTACKTOP 下的 Trapframe | env_pop_tf 中 `move sp, a0`，sp 变为新进程 env_tf 地址 |
| **Trapframe 来源** | KSTACKTOP 下的当前进程现场 | 新进程 Env 结构体中的 env_tf |
| **EPC 指向** | 异常发生时的指令（重新执行） | 新进程上次被调度离开时的 PC |
| **curenv** | 不变 | 变为新进程 |
| **地址空间** | 不变 | ASID 被重新设置 |
| **栈释放** | 隐式释放（恢复 sp 自动释放） | 旧进程栈保存在其 Env 中 |

---

## 关键设计思想

1. **Trapframe 位置约定**：内核中断栈总是在 KSTACKTOP，异常发生时现场保存在固定位置

2. **统一返回路径**：所有异常都通过 `ret_from_exception` 返回，RESTORE_ALL 只写一次

3. **调度时机**：时钟中断是抢占式调度的唯一触发点（在 lab3 阶段）

4. **进程切换的本质**：
   - 保存：把 KSTACKTOP 处的 Trapframe 拷贝到 `curenv->env_tf`
   - 恢复：把 `newenv->env_tf` 地址赋给 sp，RESTORE_ALL 自然恢复

5. **TLB 异常的特殊性**：属于"快速路径"，不涉及调度，补完页表立即返回重试

---

**一句话总结**：异常发生时，硬件把你扔到 entry.S，SAVE_ALL 把现场全存到内核栈顶；genex.S 分发找谁处理；处理完要么原封不动回去（TLB），要么趁此机会换个进程跑（时钟中断）；最后都走 RESTORE_ALL + eret 返回。
